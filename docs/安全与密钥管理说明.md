# 安全与密钥管理说明

本文从**仓库与流程**角度说明：哪些内容**不得**提交到 Git、OTA/签名场景下的**密钥**应如何管理，以及 **`zephyr_config.env`** 等本地配置的注意事项。**不替代** Zephyr/MCUboot 官方安全白皮书；量产安全需做威胁建模与独立审计。

**相关**：**[OTA与存储扩展指南.md](OTA与存储扩展指南.md)** · **[参与贡献与代码规范.md](参与贡献与代码规范.md)**

---

## 1. 禁止进入版本库的内容

| 类型 | 说明 |
|------|------|
| **私钥与证书** | OTA 签名私钥、TLS 客户端证书私钥、代码签名密钥等 |
| **密码与 Token** | Wi-Fi 密码、云 API Key、GitHub PAT、云服务凭据 |
| **本机绝对路径中的敏感信息** | 若路径暴露内部网络结构，按团队策略决定是否脱敏 |
| **含客户数据的日志 / 转储** | 调试产物勿当作文档提交 |

若确需**示例配置**，使用明显占位符（如 `YOUR_KEY_ID`、假主机名），并在旁注释「示例，勿用于生产」。

---

## 2. `zephyr_config.env` 与 `.gitignore`

- **`zephyr_config.env`** 用于填写本机 **`ZEPHYR_BASE`**、**`ZEPHYR_SDK_INSTALL_DIR`** 等，一般由 **`zephyr_config.env.template`** 复制而来。  
- 该文件通常**不应包含**密钥；若团队误把密钥写进 env，须立即**轮换密钥**并从历史中清理（必要时使用 `git filter-repo` 等工具，由管理员操作）。  
- 请确认 **`.gitignore`** 已包含 `zephyr_config.env`（或等价规则），避免推送到远程。

---

## 3. OTA / 镜像签名（MCUboot 等）

集成 **MCUboot**、**镜像签名**时常见要点：

- **私钥**仅保存在**安全介质**（HSM、专用签名机、受控 CI 密钥库），**不要**出现在应用仓库或开发者笔记本的明文路径。  
- CI 中签名：使用 **GitHub Actions / GitLab** 等提供的 **Secrets** 注入密钥材料，**不要**在 workflow 里硬编码。  
- **公钥/哈希**可随固件或分区描述分发，用于设备端校验；算法与密钥长度按产品安全等级选择。  

流程与 Kconfig 符号以当前 Zephyr 文档 **[Device Management / MCUboot](https://docs.zephyrproject.org/latest/services/device_mgmt/mcuboot.html)** 为准；本仓库 **[OTA与存储扩展指南.md](OTA与存储扩展指南.md)** 仅为架构级指引。

---

## 4. 调试与发布镜像

- **开发版**固件可打开较多日志与调试接口；**量产镜像**应关闭调试后门、收紧日志级别，并按需启用**读保护**、**安全启动**（取决于 SoC，见芯片厂商文档）。  
- 发布前核对 **[发布检查清单.md](发布检查清单.md)**，确认未误带测试密钥或调试符号策略。

---

## 5. 发现泄露时

若密钥已误提交到 Git：

1. **立即作废**该密钥并签发新密钥（视为已泄露）。  
2. 从仓库**删除**敏感文件并清理历史（需仓库管理员权限）。  
3. 通知团队轮换所有使用该密钥的系统（CI、设备端公钥更新策略等）。
