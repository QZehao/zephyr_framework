# 文档索引与阅读指南

本页是 **`docs/`** 下说明文档的**总目录**：列出全部手册、推荐阅读顺序、适用对象与概念速查，便于初学者按图索骥。

---

## 初学者从哪读起？

下面三条路径按常见需求划分，**不必**每份文档都通读；与当前任务无关的可先跳过。

### 路径 A：第一次编译本工程（零基础）

1. **[环境搭建与配置指南.md](环境搭建与配置指南.md)** — 安装 Zephyr SDK、West、CMake、Python；配置 **`zephyr_config.env`**；验证 **`west build`** 能通过。  
2. **[独立应用构建说明.md](独立应用构建说明.md)** — 理解「独立应用」目录结构、**`ZEPHYR_BASE`**、**`west build`** 常用参数；自定义板与 overlay 的约定。  
3. **[开发者入门指南.md](开发者入门指南.md)** — 项目目录、`prj.conf` / `Kconfig` 分工、添加模块与事件的最短流程。  
4. 若使用 **Nucleo L4R5** 等且遇到 **RAM 不足 / 链接溢出**，再读 **[设备树与内存配置手册.md](设备树与内存配置手册.md)**（含 **`app.overlay`** 与多段内存说明）。

### 路径 B：要改业务功能（事件、模块、IPC）

1. **[项目配置项说明.md](项目配置项说明.md)** — 所有 **`CONFIG_*`** 与 **`app_config.h`** 宏的集中释义（建议收藏，随时查）。  
2. **[事件系统详细使用说明.md](事件系统详细使用说明.md)** — 发布/订阅、队列、优先级、线程模型。  
3. **[模块系统详细使用说明.md](模块系统详细使用说明.md)** — 模块注册、生命周期、**`depends_on`** 运行时依赖。  
4. 若启用 **Thread IPC**：**[Thread_IPC服务使用说明.md](Thread_IPC服务使用说明.md)** → **[Thread_IPC模块集成指南.md](Thread_IPC模块集成指南.md)**。

### 路径 C：发布版本、对齐 CI、可选 OTA

1. **[版本管理.md](版本管理.md)** — `APP_VERSION`、Git 信息与编译时间如何进入固件。  
2. **[Zephyr版本与CI说明.md](Zephyr版本与CI说明.md)** — 本地 Zephyr 与 **`.github/workflows`** 中镜像版本如何对齐。  
3. **[发布检查清单.md](发布检查清单.md)** — 发版前检查项。  
4. 需要 **OTA / 掉电存储 / 低功耗** 方向时：**[OTA与存储扩展指南.md](OTA与存储扩展指南.md)**（与具体 SoC 强相关，需对照 Zephyr 官方文档）。

## 名词与缩写（初学者）

| 名词 | 含义 |
|------|------|
| **独立应用 / Freestanding** | 应用源码与 Zephyr 源码在不同目录；构建时依赖 **`ZEPHYR_BASE`** 找到内核。 |
| **`west`** | Zephyr 的元工具：拉代码、构建、烧录等。 |
| **`prj.conf`** | Kconfig 片段，写入 **`CONFIG_*`**，与 **`Kconfig`** 菜单一起决定最终 **`autoconf.h`**。 |
| **`app.overlay`** | 设备树覆盖文件，合并到板级 DTS；本仓库用于扩展 SRAM 等。 |
| **事件 / Event** | 模块间异步通信机制（发布–订阅），见事件系统文档。 |
| **Thread IPC（本仓库）** | 应用内多线程请求/响应服务，**不是** Zephyr 同名子系统 `IPC_SERVICE` 的全部能力。 |
