## 概述

模块系统提供了一个动态的、可扩展的组件管理框架，支持模块的注册、生命周期管理和事件驱动通信。该系统与事件系统紧密集成，为应用程序提供模块化的架构支持。

### 主要特性

| 特性 | 描述 |
|------|------|
| 最大模块数 | 16 (可配置) |
| 每模块最大事件订阅 | 8 (可配置) |
| 初始化超时 | 1000ms (可配置) |
| 优先级调度 | 支持（4 个优先级） |
| 动态注册/注销 | 支持 |
| 模块状态管理 | 完整状态机 |
| 运行时依赖排序（可选） | `depends_on` + Kconfig，见下文 |

### 模块状态机

## 应用启动与初始化顺序（Zephyr SYS_INIT）

本模板在 **`main()` 进入之前** 通过 Zephyr **`SYS_INIT(fn, POST_KERNEL, prio)`** 完成应用与子系统初始化；**同一 `POST_KERNEL` 阶段内，`prio` 数值越小越先执行**（与 `src/app/app_config.h` 中 **`APP_INIT_PRIO_*`** 宏一致）。

### 启动阶段（`main` 之前）

典型顺序为：

1. **`app_init_apply_cb`**：打印版本、清零应用控制块、写入默认 `app_config_t`（来自 `app_config.h` 宏）。
2. **`sys_log_init` / `sys_mem_init`**（受 `APP_CONFIG_ENABLE_*` 控制）。
3. **`event_system_init`** → **`event_dispatcher_init`** → **`sys_timer_init`** → **`sys_wdt_init`**（按宏开关）。
4. **`module_manager_init`**。
5. **各业务模块**在各自 **`.c`** 内通过 **`SYS_INIT(..., APP_INIT_PRIO_MODULE_*)`** 调用 **`module_manager_register()`**（示例见 `example_module_a.c`、`example_module_ipc.c` 等）。有依赖的模块应使用**更大**的 `APP_INIT_PRIO_*`（更晚注册），例如 `example_module_multi_dep` 在 A/B 之后。
6. **`app_init_finalize`**：置位「应用已初始化」、记录启动时间戳。

失败时（如 `event_dispatcher_init` 非 `EVENT_OK`）对应 init 回调返回非 0，Zephyr 启动路径会报错；具体行为以 Zephyr 版本为准。

### `app_init()` / `app_start()`（`main` 中）

- **`app_init(config)`**：在 SYS_INIT 已全部成功后返回 **`APP_OK`**；**`config` 指针当前不参与** SYS_INIT 路径（保留 API 兼容）。若需运行时覆盖配置，应在架构上另选钩子（如弱符号默认表或独立初始化模块）。
- **`app_start()`**：仍由 **`main()`** 调用：启动事件系统、分发器线程、模块管理器及已注册模块、看门狗与心跳定时器等（与改版前行为一致）。

### 新增业务模块时如何注册

1. 在模块 **`.c`** 文件末尾增加 **`SYS_INIT(your_module_auto_register, POST_KERNEL, APP_INIT_PRIO_…)`**，在回调内准备 **`config`** 并调用 **`module_manager_register(your_module_get_interface(), &cfg, &id)`**。
2. 在 **`app_config.h`** 中新增 **`APP_INIT_PRIO_MODULE_YOURS`**，取值介于 **`APP_INIT_PRIO_MODULE_MGR`** 与 **`APP_INIT_PRIO_APP_FINAL`** 之间；若依赖其它模块，**数值应大于** 被依赖模块的优先级（更晚执行）。
3. **不必**再修改 **`app_main.c`** 的集中注册列表（已移除 **`app_register_modules()`**）。

单元测试工程（**`tests/`**）不链接 **`app_main.c`**，仍可在用例中显式调用 **`module_manager_init()`** / **`module_manager_register()`**。

## 核心概念

### 模块接口（Module Interface）

所有模块必须实现的标准接口：

```c
typedef struct {
    const char *name;                     // 模块名称
    uint32_t version;                     // 版本号 (MAJOR.MINOR.PATCH)
    module_priority_t priority;           // 优先级
    const char *const *depends_on;        // 运行时依赖名列表（NULL 结尾），可选；见下文
    int (*init)(void *config);            // 初始化函数
    int (*start)(void);                   // 启动函数
    int (*stop)(void);                    // 停止函数
    int (*shutdown)(void);                // 销毁函数
    module_event_handler_t on_event;      // 事件处理回调
    module_status_t (*get_status)(void);  // 获取状态
    int (*control)(int cmd, void *arg);   // 控制命令
} module_interface_t;
```

完整定义以仓库内 **`module_base.h`** 为准。

### 模块状态

```c
typedef enum {
    MODULE_STATUS_UNINITIALIZED = 0,  // 未初始化
    MODULE_STATUS_INITIALIZING,       // 初始化中
    MODULE_STATUS_INITIALIZED,        // 已初始化
    MODULE_STATUS_RUNNING,            // 运行中
    MODULE_STATUS_STOPPED,            // 已停止
    MODULE_STATUS_ERROR,              // 错误
    MODULE_STATUS_SUSPENDED           // 挂起
} module_status_t;
```

### 模块优先级

```c
typedef enum {
    MODULE_PRIORITY_LOW      = 10,   // 低优先级（最后启动）
    MODULE_PRIORITY_NORMAL   = 5,    // 普通优先级
    MODULE_PRIORITY_HIGH     = 2,    // 高优先级
    MODULE_PRIORITY_CRITICAL = 0     // 关键优先级（最先启动）
} module_priority_t;
```

### 模块信息结构

```c
typedef struct {
    const module_interface_t *interface;      // 模块接口
    void *config;                             // 配置数据
    void *internal_data;                      // 内部数据
    module_status_t status;                   // 当前状态
    uint32_t id;                              // 模块 ID
    module_event_subscription_t event_subscriptions[8];  // 事件订阅
    uint8_t event_subscription_count;         // 订阅数量
} module_info_t;
```

## 配置选项

与模块管理器直接相关的 Kconfig 宏含义见下表；**全项目所有配置项**（含事件系统、Thread IPC 等）的集中说明见 **[项目配置项说明.md](项目配置项说明.md)**。

| 配置宏 | 含义（摘要） |
|--------|----------------|
| `CONFIG_MODULE_MANAGER` | 是否启用模块管理器 |
| `CONFIG_MAX_MODULES` | 可注册模块槽位上限 |
| `CONFIG_MODULE_INIT_TIMEOUT_MS` | `init()` 超时（注册阶段） |
| `CONFIG_MODULE_MANAGER_RUNTIME_DEPENDENCIES` | 是否按 `depends_on` 拓扑启动 / 逆序停止 |
| `CONFIG_MODULE_MANAGER_DEPENDS_LIST_MAX` | 单模块 `depends_on` 扫描上限（防死循环） |
| `CONFIG_MODULE_MANAGER_START_ALL_ABORT_ON_FAILURE` | `start_all` 是否在首次 `start` 失败后中止 |
| `CONFIG_EXAMPLE_MODULE_MULTI_DEP` | 是否编译多依赖示例模块 |

`CONFIG_MODULE_MAX_EVENT_SUBSCRIPTIONS` 在 `module_base.h` 中可通过宏覆盖（默认 8）。

`prj.conf` 示例：

```kconfig
CONFIG_MODULE_MANAGER=y
CONFIG_MAX_MODULES=16
CONFIG_MODULE_INIT_TIMEOUT_MS=1000
# CONFIG_MODULE_MANAGER_RUNTIME_DEPENDENCIES=y
```

## 运行时依赖（可选）

在 **`CONFIG_MODULE_MANAGER_RUNTIME_DEPENDENCIES=y`** 时，管理器会根据各模块 `depends_on` 指针数组（**以 NULL 结尾**，每项为其它模块的 `interface->name`）对 `module_manager_start_all()` 做拓扑排序，对 `module_manager_stop_all()` 做逆序；可与无依赖模块（`depends_on == NULL`）混在同一批。

| Kconfig | 含义 |
|---------|------|
| `MODULE_MANAGER_RUNTIME_DEPENDENCIES` | 开启运行时依赖排序 |
| `MODULE_MANAGER_DEPENDS_LIST_MAX` | **单个模块** `depends_on` 中最多扫描多少个名字（不含末尾 NULL），防止未正确终止时死循环；**不是**全系统模块总数，**不是**依赖链深度 |
| `MODULE_MANAGER_START_ALL_ABORT_ON_FAILURE` | 任一 `start` 失败后是否不再启动后续模块（利于依赖语义） |
| `EXAMPLE_MODULE_MULTI_DEP` | 编译并注册多依赖示例 `example_module_multi_dep`（依赖 `example_module_a` / `example_module_b`） |

实现要点（与 `module_manager.c` / `module_base.h` 一致）：

- 非法依赖或环会回退为仅按 `priority` 排序（停止侧为 priority 降序回退）。
- 宏参数名为 **`mod_name`**（勿用 `name`），避免与结构体字段 **`.name`** 在宏展开时冲突。
- 多依赖示例：`src/modules/example_module_multi_dep.c`。

## API 参考

### 核心 API

#### 初始化与关闭

```c
/**
 * @brief 初始化模块管理器
 * @return 0 成功，负值为错误码
 */
int module_manager_init(void);

/**
 * @brief 启动模块管理器
 * @return 0 成功，负值为错误码
 */
int module_manager_start(void);

/**
 * @brief 停止模块管理器
 * @return 0 成功，负值为错误码
 */
int module_manager_stop(void);

/**
 * @brief 关闭模块管理器（会停止并销毁所有模块）
 * @return 0 成功，负值为错误码
 */
int module_manager_shutdown(void);
```

### 模块注册 API

#### 注册与注销

```c
/**
 * @brief 注册模块
 * @param interface 模块接口指针
 * @param config 模块配置数据
 * @param module_id 输出：分配的模块 ID
 * @return 0 成功，负值为错误码
 * 
 * @note init() 在模块管理器持锁时调用，不要在 init 中调用
 *       module_manager_* API（会死锁）。如果 init 超过
 *       CONFIG_MODULE_INIT_TIMEOUT_MS，注册会失败。
 */
int module_manager_register(const module_interface_t *interface,
                            void *config,
                            uint32_t *module_id);

/**
 * @brief 注销模块
 * @param module_id 模块 ID
 * @return 0 成功，负值为错误码
 */
int module_manager_unregister(uint32_t module_id);
```

#### 模块查询

```c
/**
 * @brief 获取模块信息（线程安全快照）
 * @param module_id 模块 ID
 * @param out 输出结构体（必须非空）
 * @return 0 成功，-1 未找到或参数无效
 */
int module_manager_get_module_info(uint32_t module_id, module_info_t *out);

/**
 * @brief 通过名称获取模块 ID
 * @param name 模块名称
 * @return 模块 ID，未找到返回 0
 */
uint32_t module_manager_get_id_by_name(const char *name);

/**
 * @brief 遍历所有模块
 * @param callback 回调函数
 * @param user_data 用户数据
 * 
 * @note 不要在回调函数中调用 module_manager_* API
 */
void module_manager_foreach(void (*callback)(module_info_t *, void *),
                            void *user_data);
```

### 辅助宏

以下与 **`module_base.h`** 中实现一致；形参名为 **`mod_name`**（模块前缀），**不得**使用 `name`，否则与结构体字段 `.name` 在宏展开时冲突。

```c
// 版本号编码/解码
#define MODULE_VERSION(major, minor, patch) \
    (((major) << 16) | ((minor) << 8) | (patch))

#define MODULE_VERSION_MAJOR(v)  (((v) >> 16) & 0xFF)
#define MODULE_VERSION_MINOR(v)  (((v) >> 8) & 0xFF)
#define MODULE_VERSION_PATCH(v)  ((v) & 0xFF)

// 声明完整模块接口（需要所有回调函数）
#define DECLARE_MODULE_INTERFACE(mod_name) \
    extern const module_interface_t mod_name##_interface; \
    const module_interface_t mod_name##_interface = { \
        .name = #mod_name, \
        .version = MODULE_VERSION(1, 0, 0), \
        .priority = MODULE_PRIORITY_NORMAL, \
        .depends_on = NULL, \
        .init = mod_name##_init, \
        .start = mod_name##_start, \
        .stop = mod_name##_stop, \
        .shutdown = mod_name##_shutdown, \
        .on_event = mod_name##_on_event, \
        .get_status = mod_name##_get_status, \
        .control = mod_name##_control \
    }

#define DECLARE_MODULE_INTERFACE_WITH_DEPS(mod_name, deps_array) \
    extern const module_interface_t mod_name##_interface; \
    const module_interface_t mod_name##_interface = { \
        .name = #mod_name, \
        .version = MODULE_VERSION(1, 0, 0), \
        .priority = MODULE_PRIORITY_NORMAL, \
        .depends_on = (deps_array), \
        .init = mod_name##_init, \
        .start = mod_name##_start, \
        .stop = mod_name##_stop, \
        .shutdown = mod_name##_shutdown, \
        .on_event = mod_name##_on_event, \
        .get_status = mod_name##_get_status, \
        .control = mod_name##_control \
    }

// 声明最小模块接口（可选回调为 NULL）
#define DECLARE_MODULE_INTERFACE_MINIMAL(mod_name) \
    extern const module_interface_t mod_name##_interface; \
    const module_interface_t mod_name##_interface = { \
        .name = #mod_name, \
        .version = MODULE_VERSION(1, 0, 0), \
        .priority = MODULE_PRIORITY_NORMAL, \
        .depends_on = NULL, \
        .init = mod_name##_init, \
        .start = mod_name##_start, \
        .stop = mod_name##_stop, \
        .shutdown = NULL, \
        .on_event = mod_name##_on_event, \
        .get_status = NULL, \
        .control = NULL \
    }

#define DECLARE_MODULE_INTERFACE_MINIMAL_WITH_DEPS(mod_name, deps_array) \
    extern const module_interface_t mod_name##_interface; \
    const module_interface_t mod_name##_interface = { \
        .name = #mod_name, \
        .version = MODULE_VERSION(1, 0, 0), \
        .priority = MODULE_PRIORITY_NORMAL, \
        .depends_on = (deps_array), \
        .init = mod_name##_init, \
        .start = mod_name##_start, \
        .stop = mod_name##_stop, \
        .shutdown = NULL, \
        .on_event = mod_name##_on_event, \
        .get_status = NULL, \
        .control = NULL \
    }
```

调用方式不变，例如：`DECLARE_MODULE_INTERFACE(my_module);`、`DECLARE_MODULE_INTERFACE_MINIMAL_WITH_DEPS(foo, foo_deps);`。

## 使用指南

### 快速开始

#### 1. 创建模块

**模块头文件 (my_module.h)**:

```c
#ifndef MY_MODULE_H
#define MY_MODULE_H

#include "module_base.h"

// 模块配置结构
typedef struct {
    int param1;
    int param2;
} my_module_config_t;

// 模块接口函数声明
int my_module_init(void *config);
int my_module_start(void);
int my_module_stop(void);
int my_module_shutdown(void);
void my_module_on_event(const event_t *event, void *user_data);
```

#### 2. 注册和使用模块

**本仓库主固件**：`module_manager_init()` 与 **`module_manager_register()`** 的调用发生在 **`SYS_INIT(POST_KERNEL, …)`** 阶段（见上文「应用启动与初始化顺序」）；下列代码演示**手动顺序**，适用于理解 API 或 **`tests/`** 等不链接 `app_main.c` 的场景。

```c
#include "module_manager.h"
#include "my_module.h"

void app_main(void)
{
    // 1. 初始化模块管理器
    int ret = module_manager_init();
    if (ret != 0) {
        LOG_ERR("Failed to init module manager");
        return;
    }

    // 2. 准备模块配置
    my_module_config_t config = {
        .param1 = 10,
        .param2 = 20
    };

    // 3. 注册模块
    uint32_t module_id;
    ret = module_manager_register(&my_module_interface,
                                   &config,
                                   &module_id);
    if (ret != 0) {
        LOG_ERR("Failed to register module");
        return;
    }
    LOG_INF("Module registered with ID: %u", module_id);

    // 4. 启动模块管理器
    ret = module_manager_start();
    if (ret != 0) {
        LOG_ERR("Failed to start module manager");
        return;
    }

    // 5. 启动模块（可单独启动或启动所有）
    ret = module_manager_start_module(module_id);
    // 或：module_manager_start_all();

    // 6. 订阅事件
    ret = module_manager_subscribe(module_id, EVENT_TYPE_SENSOR_DATA);
    if (ret != 0) {
        LOG_WRN("Failed to subscribe to event");
    }

    // 7. 使用模块功能
    my_module_do_something();

    // 8. 发送事件到模块
    event_t event = {
        .type = EVENT_TYPE_GENERIC,
        .priority = EVENT_PRIORITY_NORMAL,
        .data = NULL,
        .data_len = 0
    };
    module_manager_send_to_module(module_id, &event);

    // 9. 广播事件
    module_manager_broadcast(&event);
}
```

#### 实现文件 (example_module_gpio.c)

```c
#include "example_module_gpio.h"
#include <zephyr/logging/log.h>
#include <string.h>

LOG_MODULE_REGISTER(example_module_gpio, LOG_LEVEL_INF);

// 模块私有数据
typedef struct {
    example_module_gpio_config_t config;
    module_status_t status;
    bool led_state;
    struct k_timer blink_timer;
    uint32_t blink_count;
} gpio_module_data_t;

static gpio_module_data_t g_gpio_data;

// 定时器回调
static void blink_timer_handler(struct k_timer *timer)
{
    example_module_gpio_toggle_led();
    g_gpio_data.blink_count++;
}

// 初始化
int example_module_gpio_init(void *config)
{
    LOG_INF("Initializing GPIO module");

    if (config == NULL) {
        return -EINVAL;
    }

    memcpy(&g_gpio_data.config, config, sizeof(g_gpio_data.config));
    g_gpio_data.status = MODULE_STATUS_INITIALIZING;
    g_gpio_data.led_state = false;
    g_gpio_data.blink_count = 0;

    // 初始化 LED 引脚
    if (g_gpio_data.config.led_port != NULL) {
        int ret = gpio_pin_configure(g_gpio_data.config.led_port,
                                      g_gpio_data.config.led_pin,
                                      GPIO_OUTPUT_ACTIVE);
        if (ret < 0) {
            LOG_ERR("Failed to configure LED pin");
            g_gpio_data.status = MODULE_STATUS_ERROR;
            return ret;
        }
    }

    // 初始化按钮引脚
    if (g_gpio_data.config.button_port != NULL) {
        int ret = gpio_pin_configure(g_gpio_data.config.button_port,
                                      g_gpio_data.config.button_pin,
                                      GPIO_INPUT | GPIO_PULL_UP);
        if (ret < 0) {
            LOG_ERR("Failed to configure button pin");
            g_gpio_data.status = MODULE_STATUS_ERROR;
            return ret;
        }
    }

    // 初始化定时器
    k_timer_init(&g_gpio_data.blink_timer, blink_timer_handler, NULL);

    g_gpio_data.status = MODULE_STATUS_INITIALIZED;
    LOG_INF("GPIO module initialized");
    return 0;
}

// 启动
int example_module_gpio_start(void)
{
    LOG_INF("Starting GPIO module");

    if (g_gpio_data.config.blink_interval_ms > 0) {
        k_timer_start(&g_gpio_data.blink_timer,
                      K_MSEC(g_gpio_data.config.blink_interval_ms),
                      K_MSEC(g_gpio_data.config.blink_interval_ms));
    }

    g_gpio_data.status = MODULE_STATUS_RUNNING;
    LOG_INF("GPIO module started");
    return 0;
}

// 停止
int example_module_gpio_stop(void)
{
    LOG_INF("Stopping GPIO module");

    k_timer_stop(&g_gpio_data.blink_timer);

    g_gpio_data.status = MODULE_STATUS_STOPPED;
    LOG_INF("GPIO module stopped");
    return 0;
}

// 销毁
int example_module_gpio_shutdown(void)
{
    LOG_INF("Shutting down GPIO module");

    k_timer_stop(&g_gpio_data.blink_timer);

    // 关闭 LED
    if (g_gpio_data.config.led_port != NULL) {
        gpio_pin_set(g_gpio_data.config.led_port,
                     g_gpio_data.config.led_pin, 0);
    }

    g_gpio_data.status = MODULE_STATUS_UNINITIALIZED;
    LOG_INF("GPIO module shutdown complete");
    return 0;
}

// 事件处理
void example_module_gpio_on_event(const event_t *event, void *user_data)
{
    if (event == NULL) {
        return;
    }

    LOG_DBG("GPIO module received event: type=%d", event->type);

    switch (event->type) {
    case EVENT_TYPE_SENSOR_CONFIG:
        // 处理配置事件
        if (event->data != NULL && event->data_len > 0) {
            uint8_t *data = (uint8_t *)event->data;
            // 解析并应用配置
        }
        break;

    default:
        break;
    }
}

// 获取状态
module_status_t example_module_gpio_get_status(void)
{
    return g_gpio_data.status;
}

// 控制命令
int example_module_gpio_control(int cmd, void *arg)
{
    switch (cmd) {
    case GPIO_CMD_SET_BLINK_INTERVAL:
        if (arg == NULL) {
            return -EINVAL;
        }
        g_gpio_data.config.blink_interval_ms = *(uint32_t *)arg;
        return 0;

    case GPIO_CMD_GET_BLINK_INTERVAL:
        if (arg == NULL) {
            return -EINVAL;
        }
        *(uint32_t *)arg = g_gpio_data.config.blink_interval_ms;
        return 0;

    default:
        return -ENOTSUP;
    }
}

// LED 控制
int example_module_gpio_set_led(bool on)
{
    if (g_gpio_data.status != MODULE_STATUS_RUNNING) {
        return -EPERM;
    }

    if (g_gpio_data.config.led_port != NULL) {
        gpio_pin_set(g_gpio_data.config.led_port,
                     g_gpio_data.config.led_pin, on ? 1 : 0);
        g_gpio_data.led_state = on;
        return 0;
    }
    return -ENODEV;
}

// 切换 LED 状态
bool example_module_gpio_toggle_led(void)
{
    bool new_state = !g_gpio_data.led_state;
    example_module_gpio_set_led(new_state);
    return new_state;
}

// 读取按钮
bool example_module_gpio_get_button(void)
{
    if (g_gpio_data.config.button_port != NULL) {
        return gpio_pin_get(g_gpio_data.config.button_port,
                            g_gpio_data.config.button_pin);
    }
    return false;
}

// 设置闪烁间隔
int example_module_gpio_set_blink_interval(uint32_t interval_ms)
{
    g_gpio_data.config.blink_interval_ms = interval_ms;

    if (g_gpio_data.status == MODULE_STATUS_RUNNING) {
        if (interval_ms > 0) {
            k_timer_start(&g_gpio_data.blink_timer,
                          K_MSEC(interval_ms),
                          K_MSEC(interval_ms));
        } else {
            k_timer_stop(&g_gpio_data.blink_timer);
        }
    }
    return 0;
}

// 声明模块接口
DECLARE_MODULE_INTERFACE(example_module_gpio);
```

## 故障排除

### 常见问题

#### 1. 模块注册失败

**症状**：`module_manager_register` 返回错误

**检查清单**：
- [ ] 模块接口是否正确定义？
- [ ] 是否超过最大模块数？
- [ ] init 函数是否超时？

```c
// 调试步骤
void debug_register_failure(void)
{
    module_mgr_stats_t stats;
    module_manager_get_stats(&stats);
    
    LOG_INF("Total: %u, Max: %d", 
            stats.total_modules, CONFIG_MAX_MODULES);
    
    if (stats.error_modules > 0) {
        LOG_ERR("Some modules failed to initialize");
    }
}
```

#### 2. 模块启动失败

**症状**：`module_manager_start_module` 返回错误

**可能原因**：
- 模块状态不是 INITIALIZED 或 STOPPED
- start 函数返回错误

```c
// 检查模块状态
void check_module_state(uint32_t module_id)
{
    module_info_t info;
    if (module_manager_get_module_info(module_id, &info) == 0) {
        LOG_INF("Module %s status: %d", info.interface->name, info.status);
    }
}
```

#### 3. 事件未送达模块

**症状**：模块未收到订阅的事件

**检查清单**：
- [ ] 模块是否已启动（RUNNING 状态）？
- [ ] 是否正确订阅了事件类型？
- [ ] on_event 回调是否非空？

```c
// 检查订阅状态
void check_subscription(uint32_t module_id, event_type_t event_type)
{
    module_info_t info;
    module_manager_get_module_info(module_id, &info);

    LOG_INF("Module %s subscriptions: %u",
            info.interface->name, info.event_subscription_count);

    for (uint8_t i = 0; i < info.event_subscription_count; i++) {
        if (info.event_subscriptions[i].type == event_type) {
            LOG_INF("Subscribed to event type %u", event_type);
            return;
        }
    }
    LOG_WRN("Not subscribed to event type %u", event_type);
}
```

#### 4. 死锁问题

**症状**：系统卡死

**常见原因**：在 init 函数中调用 module_manager API

```c
// ❌ 错误示例 - 会导致死锁
int bad_init(void *config)
{
    // 模块管理器持锁时调用，会死锁
    module_manager_get_id_by_name("other_module");
    return 0;
}

// ✅ 正确示例
int good_init(void *config)
{
    // 只做初始化，不调用 module_manager API
    return 0;
}
```
