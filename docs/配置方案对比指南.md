# 配置方案对比指南

本文档对比四种配置方案的内存占用和适用场景。

---

## 📊 配置方案对比表

| 配置项 | **prj.conf** (标准版) | **prj_sram.conf** (平衡版) | **prj_min.conf** (极简版) | **prj_tiny.conf** (极限版) |
|--------|----------------------|---------------------------|--------------------------|---------------------------|
| **目标内存** | ~190KB | ~40KB | ~18KB | **<10KB** |
| **适用场景** | 资源充足 MCU | 中等资源 MCU | 极受限 MCU | **极限资源 MCU** |
| **示例板型** | STM32H7/NRF52840 | STM32L4/NRF52832 | STM32F103C8T6 | **32KB SRAM 系统** |

## 📝 使用方法

### 构建标准版（默认）

```bash
west build -b mimxrt1050_fire/mimxrt1052/qspi .
# 或显式指定
west build -b mimxrt1050_fire/mimxrt1052/qspi . -DCONF_FILE="prj.conf"
```

### 构建平衡版

```bash
west build -b mimxrt1050_fire/mimxrt1052/qspi . -DCONF_FILE="prj.conf;prj_sram.conf"
```

### 构建极简版

```bash
west build -b mimxrt1050_fire/mimxrt1052/qspi . -DCONF_FILE="prj.conf;prj_min.conf"
```

### 构建极限版 ⭐ 32KB SRAM 系统

```bash
west build -b mimxrt1050_fire/mimxrt1052/qspi . -DCONF_FILE="prj.conf;prj_tiny.conf" --pristine
```

> **注意**：极限版适用于 SRAM 仅 32KB 的系统，框架占用 < 10KB，预留 22KB+ 给 APP 模块。

### 验证内存占用

```bash
# 构建后查看内存使用
west build -t mem_report

# 或查看映射文件
cat build/zephyr/zephyr.map | grep -A 5 "Memory Configuration"

# 查看 BSS 段大对象
cat build/zephyr/zephyr.map | grep "\.bss\." | grep -v "0x0"
```

## ⚠️ 功能限制说明

### 极限版限制（prj_tiny.conf）

使用 `prj_tiny.conf` 时，以下功能**不可用**：

| 功能 | 状态 | 说明 |
|------|------|------|
| Shell | ❌ 禁用 | 无交互调试 |
| Console | ❌ 禁用 | 无控制台输出 |
| Serial | ❌ 禁用 | 无串口驱动 |
| Printk | ❌ 禁用 | 无打印输出 |
| LOG | ❌ 禁用 | 无日志系统 |
| Assert | ❌ 禁用 | 无断言检查 |
| 看门狗 | ❌ 禁用 | 无看门狗保护 |
| 定时器服务 | ❌ 禁用 | 无软件定时器 |
| 内存池 | ❌ 禁用 | 无 sys_memory |
| KV 存储 | ❌ 禁用 | 无键值存储 |
| 示例模块 | ❌ 禁用 | 无示例代码 |
| 商业模块框架 | ❌ 跳过 | 条件编译跳过 |

**事件系统限制**：

| 配置项 | 标准版 | 极限版 |
|--------|--------|--------|
| 事件类型数 | 256 | **8** |
| 每类型订阅者 | 16 | **1** |
| 事件队列深度 | 64 | **4** |
| 最大模块数 | 16 | **2** |

**栈大小限制**：

| 栈类型 | 标准版 | 极限版 |
|--------|--------|--------|
| 主线程栈 | 4096 | **512** |
| ISR 栈 | 2048 | **512** |
| 空闲线程栈 | 320 | **128** |
| 分发器栈 | 2048 | **256** |
| Workqueue 栈 | 2048 | **256** |

### 极简版限制

使用 `prj_min.conf` 时，以下功能**不可用**：

- ❌ Shell 交互命令
- ❌ 内存泄漏跟踪
- ❌ 事件/分发器统计
- ❌ 日志转储命令
- ❌ 看门狗保护
- ❌ KV 存储
- ❌ 示例模块 A/B
- ❌ 日志彩色输出
- ❌ 断言检查

### 平衡版限制

使用 `prj_sram.conf` 时，以下功能**受限**：

- ⚠️ 最大 128 种事件类型（标准版 256）
- ⚠️ 每类型最多 8 个订阅者（标准版 16）
- ⚠️ 内存池仅 2 个（标准版 4 个）
- ⚠️ 最大 16 个定时器（标准版 32）
- ⚠️ 无运行时统计（`APP_ENABLE_STATS=n`）
- ⚠️ 看门狗仅软件模式

## 🔍 如何选择

### 选择标准版的场景

- ✅ MCU SRAM ≥ 256KB
- ✅ 需要完整调试功能
- ✅ 需要硬件看门狗
- ✅ 需要内存泄漏跟踪
- ✅ 需要 Shell 交互调试
- ✅ 事件类型 > 128 种

### 选择平衡版的场景

- ✅ MCU SRAM 64-128KB
- ✅ 需要基本调试功能
- ✅ 软件看门狗足够
- ✅ 事件类型 ≤ 128 种
- ✅ 订阅者 ≤ 8 个/类型
- ✅ 需要 Shell 但可放弃统计

### 选择极简版的场景

- ✅ MCU SRAM 32-64KB
- ✅ 生产环境（无需调试）
- ✅ 事件类型 ≤ 64 种
- ✅ 订阅者 ≤ 4 个/类型
- ✅ 无需交互调试
- ✅ 资源受限

### 选择极限版的场景 ⭐ 推荐

- ✅ **MCU SRAM ≤ 32KB**
- ✅ **需要预留 ≥ 22KB 给 APP 模块**
- ✅ 生产环境（完全无调试）
- ✅ 事件类型 ≤ 8 种
- ✅ 订阅者 ≤ 1 个/类型
- ✅ 无需日志、Shell、看门狗、定时器
- ✅ 极简事件驱动架构

## 🛠️ 自定义配置

如需进一步调整，可创建自己的叠加配置文件：

```bash
# 复制模板
cp prj_tiny.conf prj_custom.conf

# 编辑配置

west build -b mimxrt1050_fire/mimxrt1052/qspi . -DCONF_FILE="prj.conf;prj_custom.conf"
```

## 📌 极限配置优化技巧

### 1. 条件编译

极限配置下，以下模块会自动跳过编译：

- `sys_log.c` - 当 `CONFIG_SYS_LOG_ENABLE=n`
- `sys_memory.c` - 当 `CONFIG_SYS_MEMORY_ENABLE=n`
- `sys_watchdog.c` - 当 `CONFIG_SYS_WATCHDOG_ENABLE=n`
- `sys_timer.c` - 当 `CONFIG_SYS_TIMER_ENABLE=n`
- `example_module_*.c` - 当 `CONFIG_EXAMPLE_MODULE_*_ENABLE=n`
- 商业模块框架 - 当 `CONFIG_SIZE_OPTIMIZATIONS=y` 且无商业模块启用

### 2. 栈大小优化

栈大小是内存占用的大头，极限配置的关键优化：

```properties
# 最小可行的栈配置
CONFIG_MAIN_STACK_SIZE=512      # 主线程
CONFIG_ISR_STACK_SIZE=512       # 中断栈
CONFIG_IDLE_STACK_SIZE=128      # 空闲线程
CONFIG_EVENT_DISPATCHER_STACK_SIZE=256  # 事件分发器
CONFIG_SYSTEM_WORKQUEUE_STACK_SIZE=256  # 工作队列
```

### 3. 事件系统优化

事件系统是框架核心，合理配置可大幅节省内存：

```properties
# 极限配置
CONFIG_EVENT_MAX_TYPES=8        # 仅支持 8 种事件
CONFIG_EVENT_MAX_SUBSCRIBERS=1  # 每种事件仅 1 个订阅者
CONFIG_EVENT_QUEUE_SIZE=4       # 队列深度仅 4
```

### 4. 禁用调试输出

完全禁用输出可节省大量代码和数据空间：

```properties
CONFIG_PRINTK=n          # 禁用打印
CONFIG_CONSOLE=n         # 禁用控制台
CONFIG_SERIAL=n          # 禁用串口
CONFIG_LOG=n             # 禁用日志
CONFIG_ASSERT=n          # 禁用断言
```

## 📌 注意事项

1. **配置文件叠加顺序**：`prj.conf` → `prj_xxx.conf`（后者覆盖前者）
2. **Kconfig range 限制**：某些选项有最小值限制，见 `Kconfig` 文件
3. **内存计算**：仅包含静态分配，不含 Zephyr 内核和动态分配
4. **构建验证**：修改配置后务必 `--pristine` 重新构建
5. **实际测量**：使用 `CONFIG_INIT_STACKS=y` 可查看栈使用情况
6. **栈溢出风险**：极限栈配置可能导致复杂回调溢出，需谨慎测试

## 🔧 极限配置调优指南

### 栈大小调整原则

| 回调复杂度 | 推荐 Dispatcher 栈 | 推荐 Main 栈 |
|-----------|-------------------|-------------|
| 极简（无嵌套调用） | 256 | 512 |
| 简单（少量函数） | 512 | 1024 |
| 中等（有日志输出） | 1024 | 2048 |
| 复杂（多层嵌套） | 2048 | 4096 |

### 事件队列深度选择

| 场景 | 推荐深度 | 说明 |
|------|---------|------|
| 事件稀少（<10/秒） | 4 | 极限配置 |
| 事件中等（10-100/秒） | 8-16 | 极简配置 |
| 事件密集（>100/秒） | 32-64 | 标准配置 |
