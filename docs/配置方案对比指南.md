# 配置方案对比指南

本文档对比三种配置方案的内存占用和适用场景。

---

## 📊 配置方案对比表

| 配置项 | **prj.conf** (标准版) | **prj_sram.conf** (平衡版) | **prj_min.conf** (极简版) |
|--------|----------------------|---------------------------|--------------------------|
| **目标内存** | ~115KB | ~40KB | **~18KB** |
| **适用场景** | 资源充足 MCU | 中等资源 MCU | 极受限 MCU |
| **示例板型** | STM32H7/NRF52840 | STM32L4/NRF52832 | STM32F103C8T6 |

## 📝 使用方法

### 构建标准版（默认）

```bash
west build -b native_posix .
# 或显式指定
west build -b native_posix . -DCONF_FILE="prj.conf"
```

### 构建平衡版

```bash
west build -b native_posix . -DCONF_FILE="prj.conf;prj_sram.conf"
```

### 构建极简版

```bash
west build -b native_posix . -DCONF_FILE="prj.conf;prj_min.conf"
```

### 验证内存占用

```bash
# 构建后查看内存使用
west build -t mem_report

# 或查看映射文件
cat build/zephyr/zephyr.map | grep -A 5 "Memory Configuration"
```

## ⚠️ 功能限制说明

### 极简版限制

使用 `prj_min.conf` 时，以下功能**不可用**：

- ❌ Shell 交互命令
- ❌ 内存泄漏跟踪
- ❌ 事件/分发器统计
- ❌ 日志转储命令
- ❌ 看门狗保护
- ❌ KV 存储
- ❌ 示例模块 A/B
- ❌ 日志彩色输出
- ❌ 断言检查

### 平衡版限制

使用 `prj_sram.conf` 时，以下功能**受限**：

- ⚠️ 最大 128 种事件类型（标准版 256）
- ⚠️ 每类型最多 8 个订阅者（标准版 16）
- ⚠️ 内存池仅 2 个（标准版 4 个）
- ⚠️ 最大 16 个定时器（标准版 32）
- ⚠️ 无运行时统计（`APP_ENABLE_STATS=n`）
- ⚠️ 看门狗仅软件模式

## 🔍 如何选择

### 选择标准版的场景

- ✅ MCU SRAM ≥ 256KB
- ✅ 需要完整调试功能
- ✅ 需要硬件看门狗
- ✅ 需要内存泄漏跟踪
- ✅ 需要 Shell 交互调试
- ✅ 事件类型 > 128 种

### 选择平衡版的场景

- ✅ MCU SRAM 64-128KB
- ✅ 需要基本调试功能
- ✅ 软件看门狗足够
- ✅ 事件类型 ≤ 128 种
- ✅ 订阅者 ≤ 8 个/类型
- ✅ 需要 Shell 但可放弃统计

### 选择极简版的场景

- ✅ MCU SRAM ≤ 32KB
- ✅ 生产环境（无需调试）
- ✅ 事件类型 ≤ 64 种
- ✅ 订阅者 ≤ 4 个/类型
- ✅ 无需交互调试
- ✅ 资源极度受限

## 🛠️ 自定义配置

如需进一步调整，可创建自己的叠加配置文件：

```bash
# 复制模板
cp prj_sram.conf prj_custom.conf

# 编辑配置
vim prj_custom.conf

# 构建时使用
west build -b native_posix . -DCONF_FILE="prj.conf;prj_custom.conf"
```

### 关键配置项速查

| Kconfig 选项 | 作用 | 最小值 | 推荐值 |
|-------------|------|--------|--------|
| `CONFIG_EVENT_QUEUE_SIZE` | 事件队列深度 | 16 | 32-64 |
| `CONFIG_EVENT_MAX_SUBSCRIBERS` | 每类型订阅者数 | 4 | 8-16 |
| `CONFIG_MAX_MODULES` | 最大模块数 | 4 | 8-16 |
| `CONFIG_SYS_MEMORY_POOL_SIZE` | 内存池大小 | 1024 | 4096-8192 |
| `CONFIG_HEAP_MEM_POOL_SIZE` | Heap 大小 | 2048 | 4096-16384 |
| `CONFIG_MAIN_STACK_SIZE` | 主线程栈 | 1024 | 2048-4096 |
| `CONFIG_EVENT_DISPATCHER_STACK_SIZE` | Dispatcher 栈 | 512 | 1024-2048 |

---

## 📌 注意事项

1. **配置文件叠加顺序**：`prj.conf` → `prj_xxx.conf`（后者覆盖前者）
2. **Kconfig range 限制**：某些选项有最小值限制，见 `Kconfig` 文件
3. **内存计算**：仅包含静态分配，不含 Zephyr 内核和动态分配
4. **构建验证**：修改配置后务必构建测试，确保无编译错误
5. **实际测量**：使用 `CONFIG_INIT_STACKS=y` 可查看栈使用情况
