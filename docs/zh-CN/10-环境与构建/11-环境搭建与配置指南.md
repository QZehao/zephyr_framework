# 环境搭建与配置指南

本指南介绍如何安装 **Zephyr SDK**、**West**、**CMake**、**Python**，如何填写 **`zephyr_config.env`**，以及如何验证本工程能够 **`west build`**。**全文从零起步**；若你只需要查某一步，可用目录跳转。

**建议**：首次接触请先阅读 **[文档索引.md](../00-入门/02-文档索引.md)** 中的「路径 A：第一次编译本工程」，再对照本文逐步操作。

## 前提条件

在配置此项目之前，请确保已安装以下内容：

1. **Zephyr SDK** - 下载地址：https://github.com/zephyrproject-rtos/zephyr-sdk
2. **Zephyr 源代码** - 克隆地址：https://github.com/zephyrproject-rtos/zephyr
3. **West** - 通过以下命令安装：`pip install west`
4. **CMake**（3.20.0 或更高版本）
5. **Python 3.8+**

## 配置步骤

### 步骤 1：找到 Zephyr 安装路径

找到本地 Zephyr 安装的路径：

- **Zephyr SDK 路径**：安装 Zephyr SDK 的位置
  - Windows 默认：`C:/zephyr-sdk`
  - Linux 默认：`/opt/zephyr-sdk`
  - macOS 默认：`/opt/zephyr-sdk`

- **Zephyr 基础路径**：克隆 Zephyr 源代码的位置
  - 示例：`D:/zephyrproject/zephyr` 或 `/home/user/zephyrproject/zephyr`

### 步骤 2：配置路径

选择以下方法之一：

#### 方法 A：配置文件（推荐）

1. 复制模板文件：
   ```bash
   copy zephyr_config.env.template zephyr_config.env
   ```

2. 编辑 `zephyr_config.env`，填入您的路径：
   ```bash
   ZEPHYR_SDK_INSTALL_DIR=C:/zephyr-sdk
   ZEPHYR_BASE=D:/zephyrproject/zephyr
   ```

#### 方法 B：设置脚本

运行适合您平台的设置脚本：

```bash
# Windows PowerShell
.\scripts\setup_env.ps1

# Windows CMD
scripts\setup_env.bat

# Linux/macOS
source scripts/setup_env.sh
```

#### 方法 C：环境变量

直接设置环境变量：

```bash
# Windows (PowerShell)
$env:ZEPHYR_BASE="D:/zephyrproject/zephyr"
$env:ZEPHYR_SDK_INSTALL_DIR="C:/zephyr-sdk"

# Windows (CMD)
set ZEPHYR_BASE=D:\zephyrproject\zephyr
set ZEPHYR_SDK_INSTALL_DIR=C:\zephyr-sdk

# Linux/macOS
export ZEPHYR_BASE=/path/to/zephyr
export ZEPHYR_SDK_INSTALL_DIR=/path/to/zephyr-sdk
```

### 步骤 3：验证配置

检查路径是否正确设置：

```bash
# Windows PowerShell
echo $env:ZEPHYR_BASE
echo $env:ZEPHYR_SDK_INSTALL_DIR

# Linux/macOS
echo $ZEPHYR_BASE
echo $ZEPHYR_SDK_INSTALL_DIR
```

### 步骤 4：构建项目

```bash
# 清理构建（可选）
west build -b native_posix . --clean

# 构建 native_posix（测试）
west build -b native_posix .

# 构建目标开发板
west build -b <your_board> .
```

## 故障排除

### 错误：ZEPHYR_BASE 未设置

**解决方案**：确保设置了环境变量或运行设置脚本。

```bash
# 检查是否已设置
echo $env:ZEPHYR_BASE  # Windows PowerShell
echo $ZEPHYR_BASE      # Linux/macOS

# 如果未设置，运行设置脚本
.\scripts\setup_env.ps1  # Windows
source scripts/setup_env.sh  # Linux/macOS
```

### 错误：找不到 Zephyr SDK

**解决方案**：验证 `ZEPHYR_SDK_INSTALL_DIR` 指向正确的 SDK 位置。

```bash
# 检查目录是否存在
Test-Path "C:/zephyr-sdk"  # Windows
ls /opt/zephyr-sdk         # Linux/macOS
```

### 错误：CMake 配置失败

**解决方案**：清除 CMake 缓存并重新构建。

```bash
# 删除构建目录
rm -rf build/

# 重新构建
west build -b <your_board> . --clean
```

### 错误：找不到 West

**解决方案**：使用 pip 安装 west。

```bash
pip install west
```

## 自定义开发板配置

为特定开发板构建：

1. 在 `boards/` 目录中创建开发板配置
2. 修改 `zephyr_config.env` 设置 `DEFAULT_BOARD`
3. 使用开发板标志构建：

```bash
west build -b your_board_name .
```

## 更新路径

如果移动了 Zephyr 安装位置：

1. 用新路径编辑 `zephyr_config.env`
2. 再次运行设置脚本
3. 清理并重新构建：

```bash
west build -b <your_board> . --clean
```

## 环境变量参考

| 变量 | 描述 | 示例 |
|------|------|------|
| `ZEPHYR_BASE` | Zephyr 源代码根目录路径 | `D:/zephyrproject/zephyr` |
| `ZEPHYR_SDK_INSTALL_DIR` | Zephyr SDK 路径 | `C:/zephyr-sdk` |
| `DEFAULT_BOARD` | 默认构建开发板 | `native_posix` |
| `BUILD_DIR` | 构建输出目录 | `build` |

## Windows 用户特别说明

### PowerShell 执行策略

如果运行 `setup_env.ps1` 时遇到执行策略错误，请运行：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 路径格式

在 Windows 上，建议使用正斜杠 `/` 或双反斜杠 `\\`：
- ✅ `C:/zephyr-sdk`
- ✅ `C:\\zephyr-sdk`
- ❌ `C:\zephyr-sdk`（可能在某些情况下出现问题）

## Linux/macOS 用户特别说明

### 权限问题

如果遇到权限问题，可能需要使用 sudo 安装 SDK：

```bash
sudo ./zephyr-sdk-xxx.run
```

### 用户组

在某些 Linux 发行版上，可能需要将用户添加到特定组：

```bash
sudo usermod -a -G dialout $USER
```

## 常见问题解答

### Q: 我可以同时支持多个开发板吗？

A: 可以。只需为每个开发板分别运行 `west build -b <board>` 即可。

### Q: 如何更改日志级别？

A: 编辑 `prj.conf` 中的 `CONFIG_SYS_LOG_LEVEL` 值，或编辑 `src/app/app_config.h`。

### Q: 项目支持哪些开发板？

A: 支持所有 Zephyr 支持的开发板。在 `prj.conf` 中配置所需的外设和功能。

### Q: 如何添加新的模块？

A: 参考 `README.md` 中的"创建自定义模块"部分。
