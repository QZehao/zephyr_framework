# 烧录与调试快速指南

本文说明在 **已成功 `west build`** 的前提下，如何将固件烧录到开发板、如何查看 **串口日志**，以及如何起步 **调试**。具体命令因操作系统与调试器（ST-Link、J-Link 等）略有差异，请以 **[Zephyr 官方文档](https://docs.zephyrproject.org/latest/develop/west/build-flash-debug.html)** 为准。

**前置阅读**：[环境搭建与配置指南.md](../10-环境与构建/11-环境搭建与配置指南.md) · [独立应用构建说明.md](../10-环境与构建/12-独立应用构建说明.md)

---

## 1. 烧录（Flash）

在**应用根目录**（含 `CMakeLists.txt`）且已完成至少一次 **`west build -b <board> .`** 后：

```bash
west flash
```

- **构建目录**：若未使用默认的 **`./build`**，需显式指定，例如  
  `west flash -d build_nucleo`  
  （与当时 **`west build`** 使用的 **`-d` / `--build-dir`** 一致。  
- **切换 Runner**：默认 runner 与板卡不符时（例如改用 **J-Link**、**pyOCD**），可指定：  
  `west flash -r jlink`  
  或 `west flash -r pyocd`  
  以 **`west flash --help`** 或 Zephyr 文档中该开发板的 **Runner** 说明为准。

常用说明：

- **ST Nucleo** 等板载 ST-Link 时，通常安装 **ST 驱动** 后即可识别；若失败，检查 USB、驱动与 Zephyr 文档中该板的 **Runner** 说明。  
- 若使用 **J-Link**、**DAPLink** 等，需安装对应工具并在环境中可用（`PATH`）。  
- 多镜像 / 分区方案（如 MCUboot）需按 **[OTA与存储扩展指南.md](../70-发布与产品化/74-OTA与存储扩展指南.md)** 与官方 **MCUboot** 文档操作，**不能**仅依赖单次 `west flash` 覆盖所有分区。

## 2. 串口终端（查看 `printk` / Shell）

应用通过 **`CONFIG_SERIAL`** 与板级 **`zephyr,console`** 输出时，主机侧需用串口终端连接开发板的 **USB 虚拟串口**。

### Windows

1. 在「设备管理器」中查看 **端口 (COM 和 LPT)**，记下 **COM 编号**（如 `COM3`）。  
2. 使用 **PuTTY**、**Tera Term**，或安装 **Python pyserial** 后：

```powershell
python -m pip install pyserial
python -m serial.tools.miniterm COM3 115200
```

（将 `COM3`、`115200` 换成你的端口与波特率；本模板示例控制台常为 **115200**。）

### Linux / macOS

```bash
# 常见设备名：/dev/ttyACM0、/dev/ttyUSB0
python3 -m serial.tools.miniterm /dev/ttyACM0 115200
# 或使用 minicom、screen 等
```

权限不足时，将用户加入 **`dialout`** 等组或使用 `sudo`（不推荐长期）。

## 3. 调试（Debug）

### 3.1 `west debug`

在已配置支持 GDB/OpenOCD 的前提下：

```bash
west debug
```

会按板级默认 **runner** 启动调试会话；具体依赖 **OpenOCD**、**J-Link GDB Server** 等是否已安装。非默认构建目录时同样加 **`-d <dir>`**；需换 runner 时可用 **`west debug -r <runner>`**（与 **`west flash`** 规则类似）。

### 3.2 IDE

可使用 **VS Code**、**CLion** 等配合 **GDB** 与 Zephyr 生成的 **ELF**（通常在 **`build/zephyr/zephyr.elf`**），配置「可执行文件路径」与「GDB 服务器」；细节见 Zephyr 文档 **Debugging** 章节。

## 4. 检查清单（无输出时）

| 检查项 | 说明 |
|--------|------|
| 构建是否成功 | `build/zephyr/zephyr.bin` 或 `zephyr.elf` 存在 |
| 波特率 | 与 `prj.conf` / 板级默认一致（常 115200） |
| 控制台设备 | `zephyr,console` 对应的 UART 是否为板子 USB 所连的那一路 |
| 线序 | 若使用外接 USB-TTL，TX/RX/GND 是否与板卡丝印一致 |
