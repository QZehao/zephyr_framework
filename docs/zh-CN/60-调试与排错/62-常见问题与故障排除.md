# 常见问题与故障排除

本文汇总本仓库开发中**高频问题**与处理思路，便于初学者自助排查。若涉及板级硬件差异，仍需对照 **Zephyr 官方文档** 与芯片手册。

## 1. 环境与路径

### 1.1 `ZEPHYR_BASE not set` / 找不到 Zephyr

**现象**：CMake 报错 `ZEPHYR_BASE not set and zephyr_config.env not found`。

**处理**：

1. 将 **`zephyr_config.env.template`** 复制为 **`zephyr_config.env`**，填写本机 **`ZEPHYR_BASE`**（指向 `zephyr` 仓库根目录）与 **`ZEPHYR_SDK_INSTALL_DIR`**。  
2. 或在终端中手动设置环境变量后再执行 **`west build`**。  
3. 确认 **`CMakeLists.txt`** 会读取 `zephyr_config.env`（本工程已包含该逻辑）。

### 1.2 终端里找不到 `west`

**现象**：`'west' is not recognized`（Windows）或 `command not found: west`。

**处理**：

- 在用于 Zephyr 的 **Python 虚拟环境** 中安装：`pip install west`，并**激活该 venv** 后再执行 `west`。  
- 或将 venv 的 `Scripts`（Windows）或 `bin`（Linux/macOS）加入 **PATH**。  
- 验证：`west --version`。

### 1.3 Python / CMake 版本不满足要求

**现象**：Zephyr 或 CMake 提示版本过低。

**处理**：安装 **Python 3.10+**（以当前 Zephyr 发布说明为准）、**CMake 3.20+**，并确保命令行调用到的是新版本（`python --version`、`cmake --version`）。

## 2. 构建失败

### 2.1 `region RAM overflowed` / `noinit` 放不下

**含义**：链接阶段主 **RAM** 区域小于程序实际需要的 `.bss` / `noinit` / 堆等总和。

**处理**：

1. 阅读 **[设备树与内存配置手册.md](../40-应用开发/44-设备树与内存配置手册.md)**：确认 overlay 是否生效、主 RAM 是否已扩展到芯片允许的范围。  
   **注意**：构建 **`nucleo_l4r5zi`** 时若存在 **`boards/nucleo_l4r5zi.overlay`**，Zephyr **通常不再**自动合并根目录 **`app.overlay`**；若该板级 overlay 里只有 settings 而无 SRAM 扩展，会仍按默认约 **192KB** RAM 链接，易与 **64KB heap**、NVS 等叠加后 **RAM overflow**。本仓库已将 **640KB SRAM** 与 **settings-partition** 写在同一 **`boards/nucleo_l4r5zi.overlay`** 内。  
2. 在 **`prj.conf`** 中适当减小 **`CONFIG_HEAP_MEM_POOL_SIZE`**、各线程栈、队列深度等（见 **[项目配置项说明.md](../40-应用开发/42-项目配置项说明.md)**）。  
3. 修改 overlay 或 `prj.conf` 后建议使用 **`west build ... -p always`** 或 **`west build -t pristine`** 后重编。

### 2.2 Devicetree / overlay 解析错误

**现象**：`devicetree error: ... parse error`、**`expected node name`**。

**处理**：

- **`/delete-node/`** 必须写在 **overlay 文件顶层**，不能放在 `/ { }` 内部。  
- 自动合并的 overlay 须为 **`.overlay`** 后缀；**`.dts`** 不会作为应用 overlay 自动合并，除非用 **`DTC_OVERLAY_FILE`** 指定。  
详见 **[设备树与内存配置手册.md](../40-应用开发/44-设备树与内存配置手册.md)**。

### 2.3 `Board 'xxx' not found`

**处理**：

- 使用 Zephyr 已自带的板名（如 **`nucleo_l4r5zi`**），或确认自定义板已放在 **`BOARD_ROOT`** 对应目录且 **`CMakeLists.txt`** 已 **`list(APPEND BOARD_ROOT ...)`**。  
详见 **[独立应用构建说明.md](../10-环境与构建/12-独立应用构建说明.md)**。

### 2.4 修改配置后行为仍像旧的

**处理**：执行 **`west build -t pristine`** 或 **`west build ... -p always`**，避免 CMake/Devicetree 缓存沿用旧配置。

## 3. 运行与硬件

### 3.1 烧录成功但无串口输出

**处理**：

1. 确认 **`prj.conf`** 中已启用 **`CONFIG_SERIAL`** 及所用 UART；**`zephyr,console`** 指向的串口与开发板 **USB 转串口** 引脚一致（见板级 DTS / 用户手册）。  
2. 波特率与终端软件一致（常见 **115200**）。  
3. Windows 下在设备管理器中确认 **COM 口号**；Linux 下确认 **`/dev/ttyACM0`** 等权限。

更多步骤见 **[烧录与调试快速指南.md](61-烧录与调试快速指南.md)**。

### 3.2 单元测试无法在 PC 上运行

**处理**：本仓库测试默认目标为 **`native_posix`**，需在主机上成功配置 Zephyr 构建环境；命令与目录见 **`tests/README.md`** 与 **[单元测试与持续集成说明.md](../50-测试与CI/51-单元测试与持续集成说明.md)**。

## 4. 文档与仓库

### 4.1 旧文档文件名打不开

本仓库曾对 **`docs/`** 下部分文件重命名，旧名与新名对照见 **[文档索引.md](../00-入门/02-文档索引.md)** 末尾「旧文件名对照」表。

### 4.2 `docs/zephyr.pdf` 是什么？

若仓库中存在 **`docs/zephyr.pdf`**，一般为自行下载的 **Zephyr 官方 PDF 文档**备份，**不参与**构建；以 [Zephyr 在线文档](https://docs.zephyrproject.org/) 版本为准。
