# 5 分钟快速体验

**目标**：在 5 分钟内看到项目运行效果（无需开发板，使用 PC 模拟运行）

**适用人群**：想快速了解这个项目能做什么的人

---

## 前提条件（2 分钟）

只需确认已安装以下两项：

### 1. Python 3.8+

```bash
# 验证命令
python --version
# 或
python3 --version
```

**未安装？** → https://www.python.org/downloads/

### 2. Git

```bash
# 验证命令
git --version
```

**未安装？** → https://git-scm.com/downloads

---

## 第 1 步：克隆仓库（1 分钟）

```bash
# 克隆到本地
git clone https://github.com/your-username/zephyr_template.git
cd zephyr_template
```

---

## 第 2 步：安装 West（1 分钟）

```bash
# 安装 Zephyr 构建工具
pip install west
```

**验证安装**：
```bash
west --version
```

---

## 第 3 步：配置 Zephyr 路径（2 分钟）

### 方案 A：你已有 Zephyr 环境

```bash
# 复制模板文件
copy zephyr_config.env.template zephyr_config.env
```

然后编辑 `zephyr_config.env`，填入你的路径：

```bash
# 编辑此文件，填入你的 Zephyr 路径
ZEPHYR_SDK_INSTALL_DIR=C:/zephyr-sdk
ZEPHYR_BASE=D:/zephyrproject/zephyr
```

### 方案 B：你还没有 Zephyr 环境

**快速方案**：先跳过硬件相关功能，仅体验 `native_posix`（PC 模拟）

本项目需要 Zephyr SDK 才能完整构建。若你只想快速体验，可以：

1. **在线阅读文档**：先阅读 [文档索引.md](02-文档索引.md) 了解项目结构
2. **查看示例代码**：浏览 `src/` 目录下的源代码
3. **安装 Zephyr 后再试**：参考 [环境搭建与配置指南.md](../10-环境与构建/11-环境搭建与配置指南.md)

> 💡 **提示**：完整环境搭建约需 1-2 小时（含下载时间），建议预留充足时间。

---

## 第 4 步：构建并运行（2 分钟）

### 构建（使用 native_posix，无需开发板）

```bash
# 构建 PC 模拟版本
west build -b native_posix .
```

**成功标志**：看到以下输出
```
Built zephyr/zephyr.exe
```

### 运行

```bash
# Windows
.\build\zephyr\zephyr.exe

# Linux/macOS
./build/zephyr/zephyr
```

---

## 你看到了什么？

成功运行后，你应该看到类似输出：

```
*** Booting Zephyr OS zephyr 3.6.0 ***
[INFO] app: Application starting...
[INFO] module_manager: Module manager initialized
[INFO] event_system: Event system ready
[INFO] app: System status: OK
app> 
```

### 尝试 Shell 命令

在 `app>` 提示符后输入：

```bash
# 查看应用状态
app status

# 查看模块信息
app modules

# 查看事件统计
app events

# 查看帮助
app help
```

**示例输出**：
```
app> app status
Application: zephyr_template
Version: 1.0.0
Status: Running
Uptime: 00:00:15
```

## 项目速览

```
zephyr_template/
├── src/
│   ├── core/           # 核心事件系统
│   ├── services/       # 系统服务（日志、内存、看门狗）
│   ├── modules/        # 业务模块
│   └── app/            # 应用程序入口
├── tests/              # 单元测试
├── docs/               # 完整文档
└── scripts/            # 工具脚本
```

**核心特性**：
- 📨 **事件驱动**：模块间通过事件通信（发布 - 订阅模式）
- 🧩 **模块化**：每个功能都是独立模块，易于扩展
- 🛡️ **线程安全**：完整的同步机制
- 📝 **日志系统**：分级日志，便于调试
- 🔧 **Shell 命令**：内置调试命令
