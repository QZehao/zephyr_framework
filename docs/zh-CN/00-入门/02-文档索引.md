## 初学者从哪读起？

下面五条路径按常见需求划分，**不必**每份文档都通读；与当前任务无关的可先跳过。

### 路径 A：第一次编译本工程（零基础）

0. （若将本仓库**复制为新项目**）先看根目录 **[README.md](../../../README.md)** 中的「从本模板初始化新项目（检查清单）」与 **[04-开发者入门指南.md](04-开发者入门指南.md#从模板复制后的检查清单)**，完成 west、CMake、CI 板型等对齐。
1. **[01-5分钟快速体验.md](01-5分钟快速体验.md)** — 5 分钟在 PC 上跑起来（`native_posix`），无需开发板。
2. **[11-环境搭建与配置指南.md](../10-环境与构建/11-环境搭建与配置指南.md)** — 安装 Zephyr SDK、West、CMake、Python；配置 **`zephyr_config.env`**；验证 **`west build`** 能通过。
3. **[12-独立应用构建说明.md](../10-环境与构建/12-独立应用构建说明.md)** — 理解「独立应用」目录结构、**`ZEPHYR_BASE`**、**`west build`** 常用参数；自定义板与 overlay 的约定。
4. 如需**更换开发板**：**[13-板型迁移指南.md](../10-环境与构建/13-板型迁移指南.md)** — 完整的板型迁移流程、设备树调整、内存配置、CI 更新。
5. **[04-开发者入门指南.md](04-开发者入门指南.md)** — 项目目录、`prj.conf` / `Kconfig` 分工、添加模块与事件的最短流程。
6. 若使用 **Nucleo L4R5** 等且遇到 **RAM 不足 / 链接溢出**，再读 **[44-设备树与内存配置手册.md](../40-应用开发/44-设备树与内存配置手册.md)**（含 **`app.overlay`** 与多段内存说明）。对于 **32KB SRAM 极限场景**，参考 **[43-配置方案对比指南.md](../40-应用开发/43-配置方案对比指南.md)** 中的「极限版」配置。
7. 编译成功后要**烧录、看串口**：**[61-烧录与调试快速指南.md](../60-调试与排错/61-烧录与调试快速指南.md)**。
8. 卡住时先查 **[62-常见问题与故障排除.md](../60-调试与排错/62-常见问题与故障排除.md)**；脚本用途见 **[63-脚本与工具说明.md](../60-调试与排错/63-脚本与工具说明.md)**。

### 路径 B：要改业务功能（事件、模块、IPC）

0. （可选但推荐）**[41-Zephyr应用开发与服务指南.md](../40-应用开发/41-Zephyr应用开发与服务指南.md)** — Zephyr 通用技术：线程、同步、定时、内存、`LOG`、设备模型、ISR 注意点、服务编写模式。
1. **[42-项目配置项说明.md](../40-应用开发/42-项目配置项说明.md)** — 所有 **`CONFIG_*`** 与 **`app_config.h`** 宏的集中释义（建议收藏，随时查）。
2. **[31-事件系统详细使用说明.md](../30-核心模块/31-事件系统详细使用说明.md)** — 发布/订阅、队列、优先级、线程模型。
3. **[32-模块系统详细使用说明.md](../30-核心模块/32-模块系统详细使用说明.md)** — 模块注册、生命周期、**`depends_on`** 运行时依赖。
4. 若启用 **Thread IPC**：**[33-Thread_IPC服务使用说明.md](../30-核心模块/33-Thread_IPC服务使用说明.md)** → **[34-Thread_IPC模块集成指南.md](../30-核心模块/34-Thread_IPC模块集成指南.md)**；扩展规划见 **[35-IPC服务扩展特性规划.md](../30-核心模块/35-IPC服务扩展特性规划.md)**。
5. 使用或裁剪 **日志 / 内存池 / 定时器 / 看门狗**：**[36-系统服务使用说明.md](../30-核心模块/36-系统服务使用说明.md)**（与 **`CONFIG_SYS_*`** 对应）。

### 路径 C：发布版本、对齐 CI、可选 OTA

1. **[71-版本管理.md](../70-发布与产品化/71-版本管理.md)** — `APP_VERSION`、Git 信息与编译时间如何进入固件。
2. **[72-Zephyr版本与CI说明.md](../70-发布与产品化/72-Zephyr版本与CI说明.md)** — 本地 Zephyr 与 **`.github/workflows`** 中镜像版本如何对齐。
3. **[73-发布检查清单.md](../70-发布与产品化/73-发布检查清单.md)** — 发版前检查项。
4. 需要 **OTA / 掉电存储 / 低功耗** 方向时：**[74-OTA与存储扩展指南.md](../70-发布与产品化/74-OTA与存储扩展指南.md)**（与具体 SoC 强相关，需对照 Zephyr 官方文档）。
5. 涉及 **签名密钥、CI Secret、仓库勿提交内容**：**[75-安全与密钥管理说明.md](../70-发布与产品化/75-安全与密钥管理说明.md)**。

### 路径 D：跑单元测试、理解 CI

1. **`tests/README.md`** — ztest 目录、**`native_posix`** 运行命令、编写用例模板。
2. **[51-单元测试与持续集成说明.md](../50-测试与CI/51-单元测试与持续集成说明.md)** — 与主应用关系、GitHub Actions 分工、版本对齐提示。
3. **[52-CI平台配置保姆级手册.md](../50-测试与CI/52-CI平台配置保姆级手册.md)** — 在 **GitHub / GitLab** 上启用 CI、查看流水线、改版本与板型、排查失败的**逐步操作**。
4. 板上跑测试：**[53-硬件测试运行指南.md](../50-测试与CI/53-硬件测试运行指南.md)**；看门狗专项：**[54-watchdog_test_guide.md](../50-测试与CI/54-watchdog_test_guide.md)**。

### 路径 E：提交代码与评审

1. **[81-参与贡献与代码规范.md](../80-贡献与维护/81-参与贡献与代码规范.md)** — PR 流程、clang-format、pre-commit、CI 预期、**Commit 格式规范**。
2. 与安全相关时配合 **[75-安全与密钥管理说明.md](../70-发布与产品化/75-安全与密钥管理说明.md)**。
3. 完善文档时参考 **[82-文档改进建议.md](../80-贡献与维护/82-文档改进建议.md)**。

### 路径 F：架构学习、面试准备

1. **[21-模块化软件设计方法论.md](../20-架构设计/21-模块化软件设计方法论.md)** — 模块化设计的总体方法论。
2. **[22-模块化软件设计的详细方法.md](../20-架构设计/22-模块化软件设计的详细方法.md)** — 接口设计、依赖管理、生命周期、通信机制、分层架构。
3. **[23-框架核心技术实现细节.md](../20-架构设计/23-框架核心技术实现细节.md)** — 事件系统、模块管理器、内存管理的深度实现细节、线程安全、性能权衡、面试问题解答。
4. **[91-嵌入式AI大模型工程师学习指南.md](../90-学习资源/91-嵌入式AI大模型工程师学习指南.md)** — 嵌入式 + AI 方向的拓展学习路径。

## 名词与缩写（初学者）

| 名词 | 含义 |
|------|------|
| **独立应用 / Freestanding** | 应用源码与 Zephyr 源码在不同目录；构建时依赖 **`ZEPHYR_BASE`** 找到内核。 |
| **`west`** | Zephyr 的元工具：拉代码、构建、烧录等。 |
| **`prj.conf`** | Kconfig 片段，写入 **`CONFIG_*`**，与 **`Kconfig`** 菜单一起决定最终 **`autoconf.h`**。 |
| **`app.overlay`** | 设备树覆盖文件，合并到板级 DTS；本仓库用于扩展 SRAM 等。 |
| **事件 / Event** | 模块间异步通信机制（发布–订阅），见事件系统文档。 |
| **Thread IPC（本仓库）** | 应用内多线程请求/响应服务，**不是** Zephyr 同名子系统 `IPC_SERVICE` 的全部能力。 |
| **`k_thread` / `k_mutex`** | Zephyr 内核对象；详见 **41-Zephyr应用开发与服务指南.md**。 |
| **`DEVICE_DT_GET`** | 由 Devicetree 解析得到的设备句柄宏；配合 **`device_is_ready()`**。 |
