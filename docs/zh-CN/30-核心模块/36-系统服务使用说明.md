# 系统服务使用说明（sys_*）

本文说明本模板在 **`src/services/`** 下提供的四类系统服务：**日志**（`sys_log`）、**内存池**（`sys_memory`）、**软件定时器**（`sys_timer`）、**看门狗**（`sys_watchdog`）。它们位于 **Zephyr 内核与业务模块之间**，为事件系统、模块管理器等提供统一能力。

**相关**：**[Zephyr应用开发与服务指南.md](../40-应用开发/41-Zephyr应用开发与服务指南.md)**（Zephyr 通用内核与服务模式）· **[项目配置项说明.md](../40-应用开发/42-项目配置项说明.md)** §3 · **[开发者入门指南.md](../00-入门/04-开发者入门指南.md)** · Doxygen **`docs/api/html`**（生成方式见 **[脚本与工具说明.md](../60-调试与排错/63-脚本与工具说明.md)**）

---

## 1. 总览

| 模块 | 头文件 | 主要职责 |
|------|--------|----------|
| 日志 | `sys_log.h` / `sys_log.c` | 分级日志；**`sys_log_dest_mask_t`** 位掩码选择控制台 printk、内存环、（可选）SEGGER RTT、UART（与控制台通常共用 printk 路径） |
| 内存 | `sys_memory.h` / `sys_memory.c` | 多池分配、统计、可选分配跟踪；与 **`CONFIG_SYS_MEMORY_POOL_SIZE`** 等配合 |
| 定时器 | `sys_timer.h` / `sys_timer.c` | 基于内核定时器的一次性/周期回调封装 |
| 看门狗 | `sys_watchdog.h` / `sys_watchdog.c` | 软/硬看门狗抽象、喂狗与超时前回调 |

**初始化顺序**：主固件中各 **`sys_*_init()`** 在 **`main()` 之前**由 **`app_main.c`** 内多个 **`SYS_INIT(POST_KERNEL, APP_INIT_PRIO_*)`** 按序调用（优先级定义见 **`src/app/app_config.h`**）；**`main()`** 中 **`app_start()`** 再启动线程与模块。若某服务未初始化，对应 API 行为以实现为准（可能返回错误或未定义）。

## 1.1 最小示例（与 `app_main.c` 中 init 步骤等价）

下列片段仅演示**典型调用顺序**；宏、配置与 **`APP_CONFIG_*`** 开关以工程源码为准。完整 API 见 **`docs/api/html`**。

### 日志 `sys_log`

```c
#include "sys_log.h"

void demo_sys_log(void)
{
	const sys_log_config_t cfg = {
		.default_level = SYS_LOG_LEVEL_INF,
		.destinations = SYS_LOG_DEST_CONSOLE | SYS_LOG_DEST_MEMORY,
		.enable_timestamp = true,
		.enable_colors = true,
		.enable_module_name = true,
		.memory_buffer_size = 1024,
	};

	(void)sys_log_init(&cfg);
	sys_log_print(SYS_LOG_LEVEL_INF, "demo", "hello %d", 1);
	/* 或使用 sys_log.h 中的便捷宏 LOG_I("demo", "hello %d", 1); */
}
```

### 内存 `sys_memory`

```c
#include "sys_memory.h"

void demo_sys_mem(void)
{
	const sys_mem_config_t cfg = {
		.pool_sizes = {
			[SYS_MEM_POOL_GENERAL] = 8192,
			[SYS_MEM_POOL_EVENT] = 4096,
			[SYS_MEM_POOL_MODULE] = 4096,
			[SYS_MEM_POOL_DMA] = 0,
		},
		.enable_tracking = true,
		.enable_defrag = false,
		.max_allocations = 256,
	};

	(void)sys_mem_init(&cfg);
	void *p = sys_mem_alloc(SYS_MEM_POOL_GENERAL, 32);

	if (p != NULL) {
		sys_mem_free(SYS_MEM_POOL_GENERAL, p);
	}
}
```

### 定时器 `sys_timer`

```c
#include "sys_timer.h"

static void on_tick(sys_timer_handle_t t, void *user_data)
{
	(void)t;
	(void)user_data;
	/* 轻量逻辑；重活请投递事件或 work */
}

void demo_sys_timer(void)
{
	(void)sys_timer_init();

	const sys_timer_config_t cfg = {
		.mode = SYS_TIMER_PERIODIC,
		.delay_ms = 500,
		.period_ms = 500,
		.callback = on_tick,
		.user_data = NULL,
		.name = "demo",
		.priority = 5,
	};

	sys_timer_handle_t h = sys_timer_create(&cfg);

	if (h != NULL) {
		(void)sys_timer_start(h);
	}
}
```

### 看门狗 `sys_watchdog`

```c
#include "sys_watchdog.h"

void demo_sys_wdt(void)
{
	const wdt_config_t cfg = {
		.mode = WDT_MODE_SOFTWARE,
		.timeout_ms = 5000,
		.feed_margin_ms = 1000,
		.pre_expire_callback = NULL,
		.callback_user_data = NULL,
		.reset_on_expire = false,
		.name = "demo",
	};

	(void)sys_wdt_init(&cfg);
	(void)sys_wdt_start();
	(void)sys_wdt_feed();
}
```

## 2. 日志服务 `sys_log`

- **级别**：`sys_log_level_t`（`OFF` / `ERR` / `WRN` / `INF` / `DBG`）。
- **输出目标**：`sys_log_dest_mask_t`（按位 OR：`SYS_LOG_DEST_CONSOLE` / `SYS_LOG_DEST_MEMORY` / `SYS_LOG_DEST_RTT` / `SYS_LOG_DEST_UART`；掩码为 0 时初始化为控制台 + 内存环）。
- **配置**：`sys_log_config_t`（默认级别、是否带时间戳/颜色/模块名、内存缓冲大小等）。

### sys_log API

| 函数 | 说明 |
|------|------|
| `int sys_log_init(const sys_log_config_t *config)` | 初始化日志系统 |
| `void sys_log_set_level(const char *module, sys_log_level_t level)` | 设置模块日志级别 |
| `sys_log_level_t sys_log_get_level(const char *module)` | 获取模块日志级别 |
| `void sys_log_set_destination(sys_log_dest_mask_t dest, bool enable)` | 设置日志输出目标 |
| `void sys_log_print(sys_log_level_t level, const char *module, const char *format, ...)` | 打印日志 |
| `void sys_log_print_ts(sys_log_level_t level, const char *module, const char *format, ...)` | 带时间戳打印日志 |
| `void sys_log_hexdump(sys_log_level_t level, const char *module, const void *data, size_t len, bool ascii)` | 十六进制转储 |
| `uint32_t sys_log_get_entries(sys_log_entry_t *entries, uint32_t count, bool oldest_first)` | 获取内存环日志 |
| `void sys_log_clear_buffer(void)` | 清空内存缓冲 |
| `uint32_t sys_log_get_count(void)` | 获取已记录消息数 |
| `void sys_log_dump(sys_log_level_t level_filter)` | 转储日志到控制台 |

### sys_log 宏

| 宏 | 说明 |
|----|------|
| `LOG_E(module, fmt, ...)` | 错误级别日志 |
| `LOG_W(module, fmt, ...)` | 警告级别日志 |
| `LOG_I(module, fmt, ...)` | 信息级别日志 |
| `LOG_D(module, fmt, ...)` | 调试级别日志 |
| `LOG_HEXDUMP_E(module, data, len)` | 十六进制转储错误日志 |
| `LOG_HEXDUMP_I(module, data, len)` | 十六进制转储信息日志 |

**使用建议**：

- 业务与模块内优先使用本服务提供的宏或封装函数，便于统一格式与后续切换后端。
- 与 **`CONFIG_SYS_LOG_LEVEL`**（见项目配置项说明）对齐；发布镜像可适当提高阈值以减小日志量。

## 3. 内存服务 `sys_memory`

- **池类型**：`sys_mem_pool_type_t`（`GENERAL`、`EVENT`、`MODULE`、`DMA` 等），用于**逻辑隔离**不同用途的分配，便于统计与限流。
- **统计**：`sys_mem_stats_t`（用量、峰值、失败次数、碎片化等）。
- **调试**：可选分配跟踪（`sys_mem_alloc_info_t`），用于泄漏与重复释放分析。

### sys_memory API

| 函数 | 说明 |
|------|------|
| `int sys_mem_init(const sys_mem_config_t *config)` | 初始化内存系统（NULL 使用默认配置） |
| `void *sys_mem_alloc(sys_mem_pool_type_t type, size_t size)` | 分配内存 |
| `void *sys_mem_calloc(sys_mem_pool_type_t type, size_t size)` | 分配并清零内存 |
| `void sys_mem_free(sys_mem_pool_type_t type, void *ptr)` | 释放内存 |
| `void *sys_mem_realloc(sys_mem_pool_type_t type, void *ptr, size_t size)` | 重新分配 |
| `void sys_mem_get_stats(sys_mem_pool_type_t type, sys_mem_stats_t *stats)` | 获取池统计 |
| `void sys_mem_reset_stats(sys_mem_pool_type_t type)` | 重置统计 |
| `uint32_t sys_mem_get_active_allocations(sys_mem_pool_type_t type)` | 获取活跃分配数 |
| `void sys_mem_dump_allocations(sys_mem_pool_type_t type)` | 转储分配信息 |
| `uint32_t sys_mem_check_leaks(sys_mem_pool_type_t type)` | 检查泄漏 |
| `size_t sys_mem_defrag(sys_mem_pool_type_t type)` | 碎片整理 |
| `size_t sys_mem_get_heap_size(void)` | 获取总堆大小 |
| `size_t sys_mem_get_free_size(void)` | 获取可用堆大小 |
| `size_t sys_mem_get_min_free_size(void)` | 获取历史最小可用 |
| `void *sys_mem_alloc_with_info(sys_mem_pool_type_t type, size_t size, const char *module, uint32_t line)` | 带信息分配（调试） |

### sys_memory 宏

| 宏 | 说明 |
|----|------|
| `SYS_MEM_ALLOC_GENERAL(size)` | 从通用池分配 |
| `SYS_MEM_FREE_GENERAL(ptr)` | 释放到通用池 |
| `SYS_MEM_ALLOC_EVENT(size)` | 从事件池分配 |
| `SYS_MEM_FREE_EVENT(ptr)` | 释放到事件池 |

**使用建议**：

- 事件负载、模块私有数据等尽量从对应池分配，避免与通用池混用导致碎片化难以分析。
- **`CONFIG_SYS_MEMORY_DEBUG`** 启用时支持带信息的分配（文件名/行号）。
- **`CONFIG_SYS_MEMORY_POOL_SIZE`** 与 Zephyr **`CONFIG_HEAP_MEM_POOL_SIZE`**（`k_malloc`）是不同层次：前者为本服务管理的池，后者为内核堆；两者总和不得超过芯片 RAM 预算（参见 **[设备树与内存配置手册.md](../40-应用开发/44-设备树与内存配置手册.md)**）。

## 4. 定时器服务 `sys_timer`

- **模式**：一次性（`SYS_TIMER_ONESHOT`）与周期（`SYS_TIMER_PERIODIC`）。
- **回调**：在 **工作上下文**中执行（具体线程/中断级别以实现为准）；回调内应**避免长时间阻塞**或与事件分发器死锁。
- **句柄**：`sys_timer_handle_t` 为不透明指针，创建/启动/停止/销毁需成对使用。
- **状态**：`sys_timer_status_t`（`SYS_TIMER_STOPPED` / `SYS_TIMER_RUNNING` / `SYS_TIMER_PAUSED` / `SYS_TIMER_EXPIRED`）。

### sys_timer API

| 函数 | 说明 |
|------|------|
| `int sys_timer_init(void)` | 初始化定时器系统 |
| `sys_timer_handle_t sys_timer_create(const sys_timer_config_t *config)` | 创建定时器 |
| `int sys_timer_delete(sys_timer_handle_t timer)` | 删除定时器 |
| `int sys_timer_start(sys_timer_handle_t timer)` | 启动定时器 |
| `int sys_timer_stop(sys_timer_handle_t timer)` | 停止定时器 |
| `int sys_timer_restart(sys_timer_handle_t timer)` | 重启定时器 |
| `int sys_timer_pause(sys_timer_handle_t timer)` | 暂停定时器 |
| `int sys_timer_resume(sys_timer_handle_t timer)` | 恢复定时器 |
| `sys_timer_status_t sys_timer_get_status(sys_timer_handle_t timer)` | 获取状态 |
| `int sys_timer_set_period(sys_timer_handle_t timer, uint32_t period_ms)` | 修改周期 |
| `uint32_t sys_timer_get_time_until_expiry(sys_timer_handle_t timer)` | 获取距过期时间 |
| `int sys_timer_get_stats(sys_timer_handle_t timer, sys_timer_stats_t *stats)` | 获取统计 |
| `int sys_timer_reset_stats(sys_timer_handle_t timer)` | 重置统计 |
| `sys_timer_handle_t sys_timer_oneshot(uint32_t delay_ms, sys_timer_callback_t callback, void *user_data)` | 创建并启动一次性定时器 |
| `sys_timer_handle_t sys_timer_periodic(uint32_t period_ms, sys_timer_callback_t callback, void *user_data)` | 创建并启动周期定时器 |
| `void sys_timer_sleep(uint32_t ms)` | 睡眠（便捷封装） |
| `uint32_t sys_timer_get_uptime(void)` | 获取系统运行时间 |

**使用建议**：

- 需要与 **事件总线**联动时，在回调中仅做轻量工作，通过 **`event_publish_*`** 投递到模块线程处理。
- 精度与抖动受内核 tick、负载影响；硬实时需求需结合硬件定时器与外设文档评估。

## 5. 看门狗服务 `sys_watchdog`

- **模式**：`wdt_mode_t`（`WDT_MODE_SOFTWARE` / `WDT_MODE_HARDWARE` / `WDT_MODE_DUAL`）。
- **配置**：超时时间、喂狗余量（`feed_margin_ms`）、超时前用户回调等。
- **状态**：`wdt_status_t`（`WDT_STATUS_STOPPED` / `WDT_STATUS_RUNNING` / `WDT_STATUS_PAUSED` / `WDT_STATUS_EXPIRED` / `WDT_STATUS_ERROR`）。
- **喂狗**：主循环或健康任务中周期性调用喂狗接口；深度睡眠与低功耗场景需对照 **[OTA与存储扩展指南.md](../70-发布与产品化/74-OTA与存储扩展指南.md)** 中的 PM 说明，避免误复位。

### sys_watchdog API

| 函数 | 说明 |
|------|------|
| `int sys_wdt_init(const wdt_config_t *config)` | 初始化看门狗 |
| `int sys_wdt_start(void)` | 启动看门狗 |
| `int sys_wdt_stop(void)` | 停止看门狗 |
| `int sys_wdt_feed(void)` | 喂狗 |
| `int sys_wdt_pause(void)` | 暂停（调试用） |
| `int sys_wdt_resume(void)` | 恢复 |
| `wdt_status_t sys_wdt_get_status(void)` | 获取状态 |
| `int sys_wdt_monitor_thread(k_tid_t thread_id, const char *thread_name, uint32_t max_idle_ms)` | 注册线程监控 |
| `int sys_wdt_unmonitor_thread(k_tid_t thread_id)` | 取消线程监控 |
| `int sys_wdt_thread_alive(k_tid_t thread_id)` | 标记线程存活 |
| `void sys_wdt_get_stats(wdt_stats_t *stats)` | 获取统计 |
| `void sys_wdt_reset_stats(void)` | 重置统计 |
| `uint32_t sys_wdt_get_time_since_feed(void)` | 获取距上次喂狗时间 |
| `uint32_t sys_wdt_get_time_until_expire(void)` | 获取距过期时间 |
| `void sys_wdt_simulate_expire(void)` | 模拟过期（测试用） |

**使用建议**：

- 与 **`CONFIG_SYS_WATCHDOG_ENABLE`**、**`CONFIG_SYS_WATCHDOG_TIMEOUT_MS`** 一致。  
- 在调试阶段可临时关闭或加大超时，**量产前**恢复合理阈值。
