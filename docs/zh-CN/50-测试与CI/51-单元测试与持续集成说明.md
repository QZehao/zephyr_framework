# 单元测试与持续集成说明

本文说明本仓库 **ztest 单元测试** 的构建与运行方式，以及 **GitHub Actions / GitLab CI** 的大致分工。细节实现以 **`tests/`** 目录、**`.github/workflows/ci.yml`**、**`.gitlab-ci.yml`**（可选）为准。

**在 GitHub / GitLab 上从零启用 CI、查看流水线、改 Zephyr 版本与板型**：见 **[CI平台配置保姆级手册.md](52-CI平台配置保姆级手册.md)**。

## 1. 单元测试（`tests/`）

- **目的**：在 **主机仿真目标 `native_posix`** 上运行核心逻辑测试，无需真实硬件。
- **详细说明**（目录结构、编写用例、gcov）：请直接阅读 **`tests/README.md`**。
- **常用命令**：

```bash
west build -b native_posix tests/ --build-dir build_tests
west build -t run --build-dir build_tests
```

- **注意**：测试工程使用 **`tests/prj.conf`**，与主应用 **`prj.conf`** 分离；若修改了根目录 **`Kconfig`**，测试侧通过 **`tests/Kconfig`** 的 **`rsource`** 复用应用菜单。

### 1.1 测试覆盖范围

截至 2026-04-10，测试覆盖以下核心模块：

| 模块 | 测试文件 | 主要测试内容 |
|------|----------|--------------|
| **事件系统** | `test_event_system.c` | 初始化、类型注册、订阅/取消订阅、事件创建/释放、发布、通知、统计、队列操作 |
| **事件队列** | `test_event_queue.c` | 初始化、入队/出队、溢出策略（DROP_NEWEST/DROP_LOWEST）、清空、统计 |
| **事件分发器** | `test_event_dispatcher.c` | 初始化/启停、暂停/恢复、过滤器、手动处理、延迟统计、自定义配置 |
| **模块管理器** | `test_module_manager.c` | 注册/注销、启停所有模块、暂停/恢复、模块信息、遍历、回调、统计 |
| **系统日志** | `test_sys_log.c` | 控制台/内存日志、目标切换、环形缓冲区 |
| **系统内存** | `test_sys_memory.c` | 初始化、分配/释放、零大小分配、统计 |
| **系统定时器** | `test_sys_timer.c` | 初始化、创建/删除、单次/周期模式、统计 |
| **系统看门狗** | `test_sys_watchdog.c` | 初始化/启停、暂停/恢复、喂狗、统计、模拟过期、线程监控 |
| **IPC 服务** | `test_ipc_service.c` | 初始化、同步调用、启停 |
| **示例模块 A** | `test_example_module_a.c` | 生命周期、控制命令、事件处理、并发访问 |
| **示例模块 B** | `test_example_module_b.c` | 生命周期、通信、统计、事件处理 |
| **GPIO 模块** | `test_example_module_gpio.c` | LED 控制、按键读取、闪烁间隔、事件处理 |
| **UART 模块** | `test_example_module_uart.c` | 发送/接收、字符串/二进制、统计、事件处理 |
| **多依赖模块** | `test_example_module_multi_dep.c` | 依赖关系、版本、优先级、注册排序 |

## 2. 持续集成（GitHub Actions / GitLab）

- **GitHub**：**`.github/workflows/ci.yml`**
- **GitLab**：**`.gitlab-ci.yml`**（与 GitHub 能力对齐，见 **[CI平台配置保姆级手册.md](52-CI平台配置保姆级手册.md)**）

二者通常包含（以实际 YAML 为准）：

| 任务类型 | 典型内容 |
|----------|----------|
| **代码质量** | **`shellcheck`**（`scripts/*.sh`）、**`pre-commit run --all-files`**（含 **clang-format** 等，与 **`.pre-commit-config.yaml`** 一致） |
| **构建** | 对 **`native_posix`** 的主工程与 **`tests/`** 构建及测试运行；对 ARM 矩阵板型做编译冒烟（以 **`.github/workflows/ci.yml`** 为准） |

**版本对齐**：CI 使用的 Zephyr 容器版本见 **`Zephyr版本与CI说明.md`**；本地 **`ZEPHYR_BASE`** 建议与 CI 主版本线兼容，减少「本地通过、CI 失败」。

## 3. 与主应用的关系

- 测试与主应用**共享** `src/` 下多数实现，但**不**链接完整 **`app_main`** 与全部示例业务模块（以 **`tests/CMakeLists.txt`** 为准）。
- 新增模块或强依赖板级外设的代码时，需在测试中提供 **mock** 或条件编译，避免在 **`native_posix`** 上直接访问不存在的外设。

## 4. 测试覆盖率统计

### 4.1 生成覆盖率报告

```bash
# 构建带覆盖率的测试
west build -b native_posix tests/ --build-dir build_tests -- -DCMAKE_C_FLAGS="--coverage"

# 运行测试
west build -t run --build-dir build_tests

# 生成 HTML 报告
gcovr -r . --html --html-details build_tests/coverage.html
```

### 4.2 查看覆盖率

打开 `build_tests/coverage.html` 查看可视化报告，包含：
- 每个文件的行覆盖率
- 分支覆盖率
- 未覆盖代码行高亮

## 5. 编写新测试

### 5.1 添加测试文件

1. 在 `tests/` 目录创建 `test_<module_name>.c`
2. 在 `tests/CMakeLists.txt` 中添加源文件
3. 确保测试包含：
   - 正常路径测试
   - 边界条件测试
   - 错误处理测试
   - 生命周期测试（init → start → stop → shutdown）

### 5.2 测试模板

```c
#include <zephyr/ztest.h>
#include "your_module.h"

ZTEST(your_module, test_basic_functionality) {
    /* 测试正常功能 */
    zassert_equal(your_function(), 0, "应成功");
}

ZTEST(your_module, test_edge_cases) {
    /* 测试边界条件 */
    zassert_true(your_function_with_null() != 0, "NULL 应返回错误");
}

ZTEST(your_module, test_error_handling) {
    /* 测试错误处理 */
    zassert_equal(your_function_with_invalid_arg(), -EINVAL, "应返回 EINVAL");
}

ZTEST_SUITE(your_module, NULL, NULL, NULL, NULL, NULL);
```

### 5.3 运行单个测试

```bash
# 构建测试
west build -b native_posix tests/ --build-dir build_tests

# 编辑 tests/CMakeLists.txt，临时注释掉其他测试文件，只保留要运行的
# 然后重新构建和运行
west build -t run --build-dir build_tests
```
