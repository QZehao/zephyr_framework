## 适用场景

- 从一块开发板迁移到另一块开发板（如从 `nucleo_l4r5zi` 迁移到 `nucleo_h743zi`）
- 在项目中添加多板型支持
- 使用自定义开发板或产品板
- 更换 MCU 系列或厂商

---

## 迁移前准备

在开始之前，请准备以下信息：

1. **目标板型名称**：在 Zephyr 中的板型标识（如 `nucleo_l4r5zi`）
2. **MCU 型号**：具体的芯片型号（如 `STM32L4R5ZITx`）
3. **内存规格**：Flash 和 RAM 大小
4. **外设需求**：项目需要使用的 GPIO、UART、SPI、I2C 等外设
5. **调试接口**：SWD、JTAG 或其他调试器类型

## 步骤 1：确认 Zephyr 板型支持

### 1.1 检查 Zephyr 是否已支持目标板

```bash
# 在 Zephyr 目录下搜索板型
cd %ZEPHYR_BASE%
find boards -name "*<board_name>*"
```

或在 Zephyr 文档中查询：https://docs.zephyrproject.org/latest/boards/index.html

### 1.2 查看板型支持级别

Zephyr 对板型的支持分为以下级别：

| 级别 | 说明 |
|------|------|
| **Supported** | 完整支持，推荐用于生产 |
| **Maintained** | 有维护者支持 |
| **Provisional** | 初步支持，可能不完整 |
| **Unsupported** | 不支持，需自行移植 |

**建议**：优先选择 `Supported` 或 `Maintained` 级别的板型。

### 1.3 查阅板型文档

每个板型在 Zephyr 中都有对应的文档页面，包含：

- 板型特性与外设
- 引脚定义与功能
- 默认配置（defconfig）
- 设备树结构

示例路径：`$ZEPHYR_BASE/boards/<vendor>/<board_name>/`

## 步骤 2：添加或修改板型配置文件

### 2.1 使用 Zephyr 内置板型

如果目标板已被 Zephyr 支持，**无需**在项目中添加板型文件，直接在构建时指定即可：

```bash
west build -b <board_name> .
```

### 2.2 添加自定义板型

如需使用自定义板型（产品板或 Zephyr 未支持的板），在 `boards/` 目录下创建：

```
boards/
└── <vendor>/
    └── <board_name>/
        ├── <board_name>.yaml
        ├── <board_name>_defconfig
        ├── <board_name>.dts
        ├── <board_name>.overlay
        ├── Kconfig.defconfig
        └── <board_name>.cmake
```

#### 2.2.1 板型 YAML 文件（必需）

定义板型元数据：

```yaml
identifier: <board_name>
name: <Board Display Name>
type: mcu
arch: <arch>
toolchain:
  - gnuarmemb
  - zephyr
ram: <size_in_kb>
flash: <size_in_kb>
supported:
  - gpio
  - uart
  - spi
  - i2c
  - adc
testing:
  ignore_tags:
    - net
    - bluetooth
```

#### 2.2.2 默认配置（推荐）

`<board_name>_defconfig` 设置板型默认开启的驱动和功能：

```kconfig
CONFIG_SOC_<soc_series>_y=y
CONFIG_UART_CONSOLE=y
CONFIG_GPIO=y
```

#### 2.2.3 设备树文件（可选）

如需完全自定义设备树，创建 `.dts` 文件；否则使用设备树覆盖即可。

#### 2.2.4 CMake 配置（可选）

`<board_name>.cmake` 用于设置板型特定的构建参数：

```cmake
board_runner_args(openocd "--gdb-port=3333" "--telnet-port=4444")
include(${ZEPHYR_BASE}/boards/common/openocd.board.cmake)
```

### 2.3 启用自定义板型路径

在根目录 `CMakeLists.txt` 中取消注释 `BOARD_ROOT`：

```cmake
# 启用自定义板型目录
list(APPEND BOARD_ROOT ${CMAKE_CURRENT_SOURCE_DIR}/boards)
```

## 步骤 3：调整设备树覆盖

设备树覆盖（Overlay）用于在不修改 Zephyr 源码的情况下定制硬件配置。

### 3.1 通用覆盖文件

项目已提供以下设备树相关文件：

- `boards/overlay.dts` - **示例模板**，不会自动加载，需重命名或显式指定
- `app.overlay` - 应用级覆盖（**自动回退**，无板型专用 overlay 时生效）

### 3.2 板型特定覆盖

为特定板型创建专属覆盖文件：

```
boards/
├── overlay.dts                    # 示例模板（需改名为 <board>.overlay 才生效）
├── nucleo_l4r5zi.overlay          # 仅用于 nucleo_l4r5zi
├── nucleo_h743zi.overlay          # 仅用于 nucleo_h743zi
└── <board_name>.overlay           # 自定义板型覆盖
```

构建系统会自动根据 `BOARD` 变量加载对应的 `.overlay` 文件（如存在）。若无板型专用 overlay，则回退到 `app.overlay`。

### 3.3 常见覆盖配置

#### 3.3.1 启用 UART

```dts
&uart0 {
    status = "okay";
};
```

#### 3.3.2 配置 GPIO

```dts
&gpio0 {
    status = "okay";
};

&led0 {
    gpios = <&gpio0 1 GPIO_ACTIVE_HIGH>;
};
```

#### 3.3.3 配置 SPI/I2C

```dts
&spi0 {
    status = "okay";
    cs-gpios = <&gpio0 2 GPIO_ACTIVE_LOW>;
};

&i2c0 {
    status = "okay";
    clock-frequency = <400000>;
};
```

#### 3.3.4 配置外部晶振

```dts
&clock {
    clock-frequency = <DT_FREQ_M(8)>;  // 8MHz 外部晶振
};
```

### 3.4 完整迁移步骤（从旧板到新板）

#### 步骤 3.4.1：导出旧板型的完整设备树

```bash
# 1. 使用旧板型构建
west build -b <old_board> .

# 2. 查看编译后的完整设备树
cat build/zephyr/zephyr.dts > old_board.dts

# 3. 查看内存区域定义
cat build/zephyr/zephyr.dts | grep -A 5 "memory@"

# 4. 查看已启用的外设节点
cat build/zephyr/zephyr.dts | grep "status = \"okay\""
```

#### 步骤 3.4.2：分析新旧板型差异

| 对比项 | 旧板型 | 新板型 | 是否需要修改 |
|--------|--------|--------|--------------|
| MCU 型号 | `<old_soc>` | `<new_soc>` | - |
| Flash 大小 | `XXX KB` | `XXX KB` | 如不同需改分区 |
| RAM 大小 | `XXX KB` | `XXX KB` | 如不同需改 `zephyr,sram` |
| UART 节点 | `&uart0` | `&uart0` | 检查标签是否一致 |
| GPIO 节点 | `&gpio0` | `&gpio0` | 检查标签是否一致 |
| 外设标签 | 见导出 DTS | 见导出 DTS | 记录差异 |

#### 步骤 3.4.3：创建新板型的 overlay 文件

```bash
# 1. 复制示例模板
cp boards/overlay.dts boards/<new_board>.overlay

# 2. 或从旧板型 overlay 复制（如有）
cp boards/<old_board>.overlay boards/<new_board>.overlay
```

#### 步骤 3.4.4：修改 overlay 文件内容

```dts
/* boards/<new_board>.overlay */

/*
 * 第一步：删除 SoC 默认内存节点（如需合并 SRAM）
 * 标签名需参考新板型 SoC 的 DTSI 文件
 */
/delete-node/ &sram0;
/delete-node/ &sram1;
/delete-node/ &sram2;

/*
 * 第二步：定义连续内存区域
 * 地址和大小需查阅新板型 MCU 参考手册
 */
/ {
    sram0: memory@<base_address> {
        device_type = "memory";
        /* 基地址，大小（十六进制） */
        reg = <0x<BASE_ADDR> 0x<SIZE_IN_HEX>>;
    };
    
    /* 指定 Zephyr 使用的 SRAM 区域 */
    zephyr,sram = &sram0;
};

/*
 * 第三步：启用必要的外设
 * 标签名需与新板型 DTS 一致
 */
&uart0 {
    status = "okay";
    current-speed = <115200>;
};

&gpio0 {
    status = "okay";
};

/*
 * 第四步：定义板级外设（LED、按键等）
 */
/ {
    aliases {
        led0 = &led0;
        sw0 = &sw0;
    };

    led0: led_0 {
        gpios = <&gpio0 <PIN_NUM> GPIO_ACTIVE_HIGH>;
        label = "User LED";
    };

    sw0: sw0 {
        gpios = <&gpio0 <PIN_NUM> (GPIO_ACTIVE_LOW | GPIO_PULL_UP)>;
        label = "User Button";
    };
};

/*
 * 第五步：Flash 分区（如需 NVS/Settings/OTA）
 */
&flash0 {
    #address-cells = <1>;
    #size-cells = <1>;
    ranges = <0x0 <FLASH_BASE_ADDR> DT_SIZE_K(<FLASH_SIZE_KB>)>;
    partitions {
        ranges;
        slot0_partition: partition@0 {
            label = "image-0";
            reg = <0x0 <IMAGE_SIZE>>;
        };
        storage_partition: partition@<STORAGE_OFFSET> {
            label = "storage";
            reg = <<STORAGE_OFFSET> <STORAGE_SIZE>>;
        };
    };
};

/*
 * 第六步：绑定 Settings 分区（掉电存储）
 */
/ {
    chosen {
        zephyr,settings-partition = &storage_partition;
        zephyr,code-partition = &slot0_partition;
    };
};
```

#### 步骤 3.4.5：查找新板型的节点标签

```bash
# 方法 1：查看 Zephyr 板级 DTS 文件
cat $ZEPHYR_BASE/boards/<vendor>/<board>/<board>.dts

# 方法 2：查看 SoC 系列 DTSI 文件
cat $ZEPHYR_BASE/dts/arm/<soc_series>.dtsi

# 方法 3：构建后查看预处理后的设备树
west build -b <new_board> .
cat build/zephyr/zephyr.dts | grep -E "&(uart|gpio|spi|i2c)"

# 方法 4：使用 dtc 反编译板级 DTS
dtc -I dtb -O dts $ZEPHYR_BASE/build/<board>/zephyr/zephyr.dts -o board.dts
```

#### 步骤 3.4.6：常见 MCU 内存标签参考

| MCU 系列 | SRAM 标签 | Flash 标签 | 示例 |
|----------|-----------|------------|------|
| STM32L4 | `sram0`, `sram1`, `sram2` | `flash0` | Nucleo L4R5ZI |
| STM32H7 | `sram0`, `sram1`, `sram2`, `sram3` | `flash0` | Nucleo H743ZI |
| STM32F4 | `sram0` | `flash0` | Nucleo F429ZI |
| nRF52 | `sram0` | `flash0` | nRF52840 DK |
| nRF53 | `sram0`, `sram1` | `flash0` | nRF5340 DK |
| RP2040 | `sram0` | `flash0` | Raspberry Pi Pico |

### 3.5 验证设备树

```bash
# 查看最终合并后的设备树
west build -b <board_name>
cat build/zephyr/zephyr.dts

# 检查内存区域是否正确
cat build/zephyr/zephyr.dts | grep -A 3 "memory@"

# 检查 zephyr,sram 指向
cat build/zephyr/zephyr.dts | grep "zephyr,sram"

# 检查外设状态
cat build/zephyr/zephyr.dts | grep "status = \"okay\""
```

## 步骤 4：调整内存与分区配置

### 4.1 检查 RAM/Flash 大小

不同板型的内存规格可能差异很大，需调整以下配置：

```kconfig
# prj.conf 或板型 defconfig
CONFIG_RAM_SIZE=<size_in_kb>
CONFIG_FLASH_SIZE=<size_in_kb>
```

### 4.2 完整内存配置迁移步骤

#### 步骤 4.2.1：查阅新板型内存规格

| 信息项 | 获取方式 | 示例值 |
|--------|----------|--------|
| SRAM 总量 | MCU 数据手册 | 640 KB |
| SRAM 分区 | MCU 参考手册 | SRAM1: 192KB, SRAM2: 64KB, SRAM3: 384KB |
| Flash 总量 | MCU 数据手册 | 2048 KB |
| SRAM 基地址 | MCU 参考手册 | 0x20000000 |
| Flash 基地址 | MCU 参考手册 | 0x08000000 |

#### 步骤 4.2.2：计算可用内存

```
总 SRAM = SRAM1 + SRAM2 + SRAM3 + ...

可用主 RAM（zephyr,sram）= 连续映射的 SRAM 总量
保留 SRAM（可选）= 用于特殊用途（DMA、备份等）

可用 Heap = 总 SRAM - 主 RAM 占用 - 保留 SRAM
```

#### 步骤 4.2.3：调整堆大小

根据可用 RAM 调整内核堆：

```kconfig
# 大 RAM 板型（>256KB）
CONFIG_HEAP_MEM_POOL_SIZE=65536    # 64KB

# 中等 RAM 板型（128-256KB）
CONFIG_HEAP_MEM_POOL_SIZE=32768    # 32KB

# 小 RAM 板型（<64KB）
CONFIG_HEAP_MEM_POOL_SIZE=8192     # 8KB
```

#### 步骤 4.2.4：调整栈大小

```kconfig
# 大 RAM 板型
CONFIG_MAIN_STACK_SIZE=4096
CONFIG_SYSTEM_WORKQUEUE_STACK_SIZE=2048
CONFIG_EVENT_DISPATCHER_STACK_SIZE=4096

# 小 RAM 板型
CONFIG_MAIN_STACK_SIZE=2048
CONFIG_SYSTEM_WORKQUEUE_STACK_SIZE=1024
CONFIG_EVENT_DISPATCHER_STACK_SIZE=2048
```

#### 步骤 4.2.5：内存配置迁移示例

**示例 A：从 STM32L4R5ZI (640KB RAM) 迁移到 STM32H743ZI (1MB RAM)**

```dts
/* boards/nucleo_h743zi.overlay */

/* H743ZI 有 SRAM1(128KB) + SRAM2(128KB) + SRAM3(384KB) + SRAM4(64KB) + AXI SRAM(320KB) */
/* 地址连续映射，可合并为 1MB 使用 */

/delete-node/ &sram0;
/delete-node/ &sram1;
/delete-node/ &sram2;
/delete-node/ &sram3;

/ {
    sram0: memory@24000000 {
        device_type = "memory";
        /* H743ZI: 1MB = 0x100000 */
        reg = <0x24000000 0x100000>;
    };
    
    zephyr,sram = &sram0;
};
```

```kconfig
# prj.conf 或 prj_h743zi.conf
CONFIG_HEAP_MEM_POOL_SIZE=131072   # 128KB（更大堆）
CONFIG_MAIN_STACK_SIZE=8192        # 8KB（更宽松）
CONFIG_EVENT_DISPATCHER_STACK_SIZE=8192
```

**示例 B：从 STM32L4R5ZI (640KB RAM) 迁移到 nRF52840 (256KB RAM)**

```dts
/* boards/nrf52840dk.overlay */

/* nRF52840: 256KB RAM 连续 */
/delete-node/ &sram0;

/ {
    sram0: memory@20000000 {
        device_type = "memory";
        /* 256KB = 0x40000 */
        reg = <0x20000000 0x40000>;
    };
    
    zephyr,sram = &sram0;
};
```

```kconfig
# prj.conf 或 prj_nrf52840.conf
CONFIG_HEAP_MEM_POOL_SIZE=32768    # 32KB（减小）
CONFIG_MAIN_STACK_SIZE=2048        # 2KB
CONFIG_SYSTEM_WORKQUEUE_STACK_SIZE=1024
CONFIG_EVENT_DISPATCHER_STACK_SIZE=2048

# 可选：关闭日志以节省资源
# CONFIG_LOG=n
```

**示例 C：小 RAM 板型（如 STM32F103C8T6 - 20KB RAM）**

```dts
/* boards/bluepill.overlay */

/* STM32F103C8T6: 20KB RAM */
/delete-node/ &sram0;

/ {
    sram0: memory@20000000 {
        device_type = "memory";
        /* 20KB = 0x5000 */
        reg = <0x20000000 0x5000>;
    };
    
    zephyr,sram = &sram0;
};
```

```kconfig
# prj_bluepill.conf - 精简配置
CONFIG_HEAP_MEM_POOL_SIZE=4096     # 4KB（最小）
CONFIG_MAIN_STACK_SIZE=1024        # 1KB
CONFIG_SYSTEM_WORKQUEUE_STACK_SIZE=512
CONFIG_EVENT_DISPATCHER_STACK_SIZE=1024

# 关闭不必要的功能
CONFIG_LOG=n
CONFIG_PRINTK=n
CONFIG_CONSOLE=n
CONFIG_SYS_LOG_LEVEL=0

# 禁用复杂服务
CONFIG_SYS_WATCHDOG=n
CONFIG_SYS_TIMER=n
CONFIG_THREAD_IPC_SERVICE=n
```

### 4.3 Flash 分区（如需 NVS/OTA）

在设备树覆盖中定义 Flash 分区：

```dts
/* 根据新板型 Flash 大小调整分区 */
&flash0 {
    #address-cells = <1>;
    #size-cells = <1>;
    ranges = <0x0 <FLASH_BASE> DT_SIZE_K(<FLASH_SIZE_KB>)>;
    partitions {
        ranges;
        slot0_partition: partition@0 {
            label = "image-0";
            /* 预留 80% 给应用 */
            reg = <0x0 (DT_SIZE_K(<FLASH_SIZE_KB>) * 80 / 100)>;
        };
        storage_partition: partition@<storage_offset> {
            label = "storage";
            /* 剩余 20% 给存储 */
            reg = <<storage_offset> (DT_SIZE_K(<FLASH_SIZE_KB>) * 20 / 100)>;
        };
    };
};

/ {
    chosen {
        zephyr,code-partition = &slot0_partition;
        zephyr,settings-partition = &storage_partition;
    };
};
```

### 4.4 使用 SRAM 分区配置

本项目使用 SRAM 分区优化，更换板型时需检查：

1. 查看 `prj.conf` 中的 SRAM 配置
2. 检查板型 overlay 是否有 SRAM 合并配置
3. 小 RAM 板型可能需要关闭部分功能

### 4.5 内存配置检查清单

- [ ] 已查阅新板型 MCU 的内存规格（数据手册）
- [ ] 已计算可用 RAM 总量
- [ ] 已调整 `CONFIG_HEAP_MEM_POOL_SIZE`（不超过可用 RAM 的 50%）
- [ ] 已调整各栈大小（`MAIN_STACK_SIZE`、`EVENT_DISPATCHER_STACK_SIZE` 等）
- [ ] 已在 overlay 中定义正确的 `zephyr,sram` 区域
- [ ] 已定义 Flash 分区（如需掉电存储）
- [ ] 小 RAM 板型已关闭不必要的功能（日志、复杂服务）
- [ ] 编译后检查内存使用率（RAM < 80% 为安全）
- [ ] 已验证运行时不会发生栈溢出或内存不足

## 步骤 5：更新项目配置文件

### 5.1 更新 prj.conf

根据新板型的硬件能力调整 `prj.conf`：

```kconfig
# 关闭不需要的功能以节省资源
CONFIG_LOG=n                    # 小 RAM 板型可关闭日志
CONFIG_SYS_LOG_LEVEL=0

# 或启用特定硬件驱动
CONFIG_GPIO=y
CONFIG_SPI=y
CONFIG_I2C=y
CONFIG_ADC=y
```

### 5.2 创建板型专用配置文件

为不同板型创建专用配置文件：

```
prj.conf                        # 默认配置
prj_nucleo_l4r5zi.conf          # L4 系列配置
prj_nucleo_h743zi.conf          # H7 系列配置（高性能）
prj_low_ram.conf                # 小 RAM 板型精简配置
```

构建时指定：

```bash
west build -b <board_name> -DCONF_FILE="prj.conf;prj_<board_name>.conf" .
```

### 5.3 调整模块依赖

某些模块可能依赖特定硬件，需在 `app_config.h` 中调整：

```c
// 根据板型启用/禁用模块
#if defined(CONFIG_BOARD_NUCLEO_L4R5ZI)
    #define APP_INIT_PRIO_MODULE_GPIO  85
#elif defined(CONFIG_BOARD_NUCLEO_H743ZI)
    #define APP_INIT_PRIO_MODULE_GPIO  85
    #define ENABLE_HIGH_PERF_MODULE    1
#else
    #define ENABLE_HIGH_PERF_MODULE    0
#endif
```

## 常见问题

### Q1: 如何查找板型的准确名称？

```bash
# 在 Zephyr 目录下
cd %ZEPHYR_BASE%
west boards | grep <keyword>
```

### Q2: 自定义板型不被识别？

确保：
1. `CMakeLists.txt` 中启用了 `BOARD_ROOT`
2. 板型目录结构正确
3. `.yaml` 文件中的 `identifier` 与板型名一致

### Q3: 如何迁移现有配置到新板型？

1. 复制当前 `prj.conf` 和 `.overlay` 文件
2. 根据新板型规格调整内存配置
3. 修改设备树中外设的引脚定义
4. 逐步启用功能，验证稳定性

### Q4: 多板型如何管理配置？

推荐使用配置组合：

```
prj.conf                    # 公共基础配置
prj_common.overlay          # 公共设备树覆盖
prj_<board_name>.conf       # 板型特定配置
boards/<board_name>.overlay # 板型特定覆盖
```

构建时组合：

```bash
west build -b <board_name> -DCONF_FILE="prj.conf;prj_<board_name>.conf" .
```

### Q5: 如何验证板型迁移是否成功？

完成以下检查清单：

- [ ] 构建无编译错误
- [ ] 内存使用在板型限制内
- [ ] 烧录成功
- [ ] 串口有输出
- [ ] 基本功能（GPIO、UART）正常工作
- [ ] 项目特定功能正常
- [ ] CI/CD 通过

## 附录 A：完整迁移实战示例

### A.1 场景：从 Nucleo L4R5ZI 迁移到 Nucleo H743ZI

**背景**：项目原使用 `nucleo_l4r5zi`（640KB RAM, 2MB Flash），现需迁移到 `nucleo_h743zi`（1MB RAM, 2MB Flash）。

#### 步骤 1：导出旧板型设备树

```bash
# 使用旧板型构建
west build -b nucleo_l4r5zi .

# 导出完整设备树
cat build/zephyr/zephyr.dts > nucleo_l4r5zi_full.dts

# 查看关键信息
cat nucleo_l4r5zi_full.dts | grep -A 3 "memory@"
cat nucleo_l4r5zi_full.dts | grep "zephyr,sram"
```

#### 步骤 2：查阅 H743ZI 规格

| 参数 | L4R5ZI | H743ZI |
|------|--------|--------|
| SRAM | 640KB | 1MB (1024KB) |
| Flash | 2MB | 2MB |
| SRAM 基址 | 0x20000000 | 0x24000000 |
| Flash 基址 | 0x08000000 | 0x08000000 |

#### 步骤 3：创建新板型 overlay

```bash
# 复制模板
cp boards/overlay.dts boards/nucleo_h743zi.overlay
```

编辑 `boards/nucleo_h743zi.overlay`：

```dts
/* boards/nucleo_h743zi.overlay */
/* STM32H743ZI: 1MB SRAM, 2MB Flash */

/* 删除 SoC 默认内存节点 */
/delete-node/ &sram0;
/delete-node/ &sram1;
/delete-node/ &sram2;
/delete-node/ &sram3;

/ {
    /* 定义 1MB 连续 SRAM */
    sram0: memory@24000000 {
        device_type = "memory";
        reg = <0x24000000 0x100000>;  /* 1MB = 0x100000 */
    };
    
    zephyr,sram = &sram0;
};

/* Flash 分区（2MB） */
&flash0 {
    #address-cells = <1>;
    #size-cells = <1>;
    ranges = <0x0 0x08000000 DT_SIZE_K(2048)>;
    partitions {
        ranges;
        slot0_partition: partition@0 {
            label = "image-0";
            reg = <0x0 0x001C0000>;  /* ~1.75MB for app */
        };
        storage_partition: partition@1c0000 {
            label = "storage";
            reg = <0x001C0000 0x00040000>;  /* 256KB for settings */
        };
    };
};

/ {
    chosen {
        zephyr,settings-partition = &storage_partition;
        zephyr,code-partition = &slot0_partition;
    };
};

/* 启用外设 */
&uart0 {
    status = "okay";
    current-speed = <115200>;
};

&gpio0 {
    status = "okay";
};
```

#### 步骤 4：创建板型专用配置文件

创建 `prj_nucleo_h743zi.conf`：

```kconfig
# prj_nucleo_h743zi.conf
# H743ZI 性能更强，RAM 更大，可适当增加资源

# 增大 Heap 到 128KB
CONFIG_HEAP_MEM_POOL_SIZE=131072

# 增大主栈和工作队列栈
CONFIG_MAIN_STACK_SIZE=8192
CONFIG_SYSTEM_WORKQUEUE_STACK_SIZE=4096
CONFIG_EVENT_DISPATCHER_STACK_SIZE=8192

# H7 系列日志可更详细
CONFIG_SYS_LOG_LEVEL=4
CONFIG_LOG_DEFAULT_LEVEL=4
```

#### 步骤 5：构建测试

```bash
# 清理旧构建
west build -t pristine

# 使用新板型和配置文件构建
west build -b nucleo_h743zi \
    -DCONF_FILE="prj.conf;prj_nucleo_h743zi.conf" .

# 查看内存使用
arm-none-eabi-size build/zephyr/zephyr.elf

# 查看设备树
cat build/zephyr/zephyr.dts | grep -A 3 "memory@"
```

```bash

west flash

# 串口监控
west console
```

### A.2 场景：从 Nucleo L4R5ZI 迁移到 nRF52840 DK

**背景**：从 STM32 迁移到 Nordic nRF52840（256KB RAM, 1MB Flash），资源更紧张。

#### 步骤 1：创建精简 overlay

```dts
/* boards/nrf52840dk.overlay */
/* nRF52840: 256KB SRAM, 1MB Flash */

/delete-node/ &sram0;

/ {
    sram0: memory@20000000 {
        device_type = "memory";
        reg = <0x20000000 0x40000>;  /* 256KB = 0x40000 */
    };
    
    zephyr,sram = &sram0;
};

/* Flash 分区（1MB） */
&flash0 {
    #address-cells = <1>;
    #size-cells = <1>;
    ranges = <0x0 0x00000000 DT_SIZE_K(1024)>;
    partitions {
        ranges;
        slot0_partition: partition@0 {
            label = "image-0";
            reg = <0x0 0x000C0000>;  /* 768KB for app */
        };
        storage_partition: partition@c0000 {
            label = "storage";
            reg = <0x000C0000 0x00040000>;  /* 256KB for settings */
        };
    };
};

/ {
    chosen {
        zephyr,settings-partition = &storage_partition;
        zephyr,code-partition = &slot0_partition;
    };
};

&uart0 {
    status = "okay";
    current-speed = <115200>;
};
```

#### 步骤 2：创建精简配置

创建 `prj_nrf52840.conf`：

```kconfig
# prj_nrf52840.conf
# nRF52840 RAM 较小，需精简配置

# 减小 Heap 到 32KB
CONFIG_HEAP_MEM_POOL_SIZE=32768

# 减小栈大小
CONFIG_MAIN_STACK_SIZE=2048
CONFIG_SYSTEM_WORKQUEUE_STACK_SIZE=1024
CONFIG_EVENT_DISPATCHER_STACK_SIZE=2048

# 关闭调试日志
CONFIG_LOG=n
CONFIG_SYS_LOG_LEVEL=0

# 可选：关闭复杂服务
# CONFIG_THREAD_IPC_SERVICE=n
# CONFIG_SYS_WATCHDOG=n
```

#### 步骤 3：构建测试

```bash
west build -t pristine
west build -b nrf52840dk_nrf52840 \
    -DCONF_FILE="prj.conf;prj_nrf52840.conf" .

# 检查 RAM 使用
arm-none-eabi-size build/zephyr/zephyr.elf
```
