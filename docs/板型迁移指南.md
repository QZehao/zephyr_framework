## 适用场景

- 从一块开发板迁移到另一块开发板（如从 `nucleo_l4r5zi` 迁移到 `nucleo_h743zi`）
- 在项目中添加多板型支持
- 使用自定义开发板或产品板
- 更换 MCU 系列或厂商

---

## 迁移前准备

在开始之前，请准备以下信息：

1. **目标板型名称**：在 Zephyr 中的板型标识（如 `nucleo_l4r5zi`）
2. **MCU 型号**：具体的芯片型号（如 `STM32L4R5ZITx`）
3. **内存规格**：Flash 和 RAM 大小
4. **外设需求**：项目需要使用的 GPIO、UART、SPI、I2C 等外设
5. **调试接口**：SWD、JTAG 或其他调试器类型

## 步骤 1：确认 Zephyr 板型支持

### 1.1 检查 Zephyr 是否已支持目标板

```bash
# 在 Zephyr 目录下搜索板型
cd %ZEPHYR_BASE%
find boards -name "*<board_name>*"
```

或在 Zephyr 文档中查询：https://docs.zephyrproject.org/latest/boards/index.html

### 1.2 查看板型支持级别

Zephyr 对板型的支持分为以下级别：

| 级别 | 说明 |
|------|------|
| **Supported** | 完整支持，推荐用于生产 |
| **Maintained** | 有维护者支持 |
| **Provisional** | 初步支持，可能不完整 |
| **Unsupported** | 不支持，需自行移植 |

**建议**：优先选择 `Supported` 或 `Maintained` 级别的板型。

### 1.3 查阅板型文档

每个板型在 Zephyr 中都有对应的文档页面，包含：

- 板型特性与外设
- 引脚定义与功能
- 默认配置（defconfig）
- 设备树结构

示例路径：`$ZEPHYR_BASE/boards/<vendor>/<board_name>/`

## 步骤 2：添加或修改板型配置文件

### 2.1 使用 Zephyr 内置板型

如果目标板已被 Zephyr 支持，**无需**在项目中添加板型文件，直接在构建时指定即可：

```bash
west build -b <board_name> .
```

### 2.2 添加自定义板型

如需使用自定义板型（产品板或 Zephyr 未支持的板），在 `boards/` 目录下创建：

```
boards/
└── <vendor>/
    └── <board_name>/
        ├── <board_name>.yaml
        ├── <board_name>_defconfig
        ├── <board_name>.dts
        ├── <board_name>.overlay
        ├── Kconfig.defconfig
        └── <board_name>.cmake
```

#### 2.2.1 板型 YAML 文件（必需）

定义板型元数据：

```yaml
identifier: <board_name>
name: <Board Display Name>
type: mcu
arch: <arch>
toolchain:
  - gnuarmemb
  - zephyr
ram: <size_in_kb>
flash: <size_in_kb>
supported:
  - gpio
  - uart
  - spi
  - i2c
  - adc
testing:
  ignore_tags:
    - net
    - bluetooth
```

#### 2.2.2 默认配置（推荐）

`<board_name>_defconfig` 设置板型默认开启的驱动和功能：

```kconfig
CONFIG_SOC_<soc_series>_y=y
CONFIG_UART_CONSOLE=y
CONFIG_GPIO=y
```

#### 2.2.3 设备树文件（可选）

如需完全自定义设备树，创建 `.dts` 文件；否则使用设备树覆盖即可。

#### 2.2.4 CMake 配置（可选）

`<board_name>.cmake` 用于设置板型特定的构建参数：

```cmake
board_runner_args(openocd "--gdb-port=3333" "--telnet-port=4444")
include(${ZEPHYR_BASE}/boards/common/openocd.board.cmake)
```

### 2.3 启用自定义板型路径

在根目录 `CMakeLists.txt` 中取消注释 `BOARD_ROOT`：

```cmake
# 启用自定义板型目录
list(APPEND BOARD_ROOT ${CMAKE_CURRENT_SOURCE_DIR}/boards)
```

## 步骤 3：调整设备树覆盖

设备树覆盖（Overlay）用于在不修改 Zephyr 源码的情况下定制硬件配置。

### 3.1 通用覆盖文件

项目已提供通用覆盖文件：

- `boards/overlay.dts` - 通用设备树覆盖
- `app.overlay` - 应用级覆盖（会被自动合并）

### 3.2 板型特定覆盖

为特定板型创建专属覆盖文件：

```
boards/
├── overlay.dts                    # 通用覆盖
├── nucleo_l4r5zi.overlay          # 仅用于 nucleo_l4r5zi
├── nucleo_h743zi.overlay          # 仅用于 nucleo_h743zi
└── <board_name>.overlay           # 自定义板型覆盖
```

构建系统会自动根据 `BOARD` 变量加载对应的 `.overlay` 文件。

### 3.3 常见覆盖配置

#### 3.3.1 启用 UART

```dts
&uart0 {
    status = "okay";
};
```

#### 3.3.2 配置 GPIO

```dts
&gpio0 {
    status = "okay";
};

&led0 {
    gpios = <&gpio0 1 GPIO_ACTIVE_HIGH>;
};
```

#### 3.3.3 配置 SPI/I2C

```dts
&spi0 {
    status = "okay";
    cs-gpios = <&gpio0 2 GPIO_ACTIVE_LOW>;
};

&i2c0 {
    status = "okay";
    clock-frequency = <400000>;
};
```

#### 3.3.4 配置外部晶振

```dts
&clock {
    clock-frequency = <DT_FREQ_M(8)>;  // 8MHz 外部晶振
};
```

### 3.4 验证设备树

```bash
# 查看最终合并后的设备树
west build -b <board_name>
cat build/zephyr/zephyr.dts
```

## 步骤 4：调整内存与分区配置

### 4.1 检查 RAM/Flash 大小

不同板型的内存规格可能差异很大，需调整以下配置：

```kconfig
# prj.conf 或板型 defconfig
CONFIG_RAM_SIZE=<size_in_kb>
CONFIG_FLASH_SIZE=<size_in_kb>
```

### 4.2 调整堆大小

根据可用 RAM 调整内核堆：

```kconfig
# 根据板型 RAM 大小调整（小 RAM 板型需减小）
CONFIG_HEAP_MEM_POOL_SIZE=32768    # 32KB
```

### 4.3 调整栈大小

```kconfig
# 小 RAM 板型需减小栈大小
CONFIG_MAIN_STACK_SIZE=2048
CONFIG_SYSTEM_WORKQUEUE_STACK_SIZE=1024
CONFIG_EVENT_DISPATCHER_STACK_SIZE=2048
```

### 4.4 Flash 分区（如需 NVS/OTA）

在设备树覆盖中定义 Flash 分区：

```dts
/ {
    chosen {
        zephyr,code-partition = &flash_partition_0;
        zephyr,storage-partition = &storage_partition;
    };

    flash_partition_0: partition@0 {
        label = "MCUBoot";
        reg = <0x08000000 0x8000>;
    };

    storage_partition: partition@80000 {
        label = "Storage";
        reg = <0x08080000 0x20000>;
    };
};
```

### 4.5 使用 SRAM 分区配置

本项目使用 SRAM 分区优化，更换板型时需检查：

1. 查看 `prj.conf` 中的 SRAM 配置
2. 检查板型 overlay 是否有 SRAM 合并配置
3. 小 RAM 板型可能需要关闭部分功能

## 步骤 5：更新项目配置文件

### 5.1 更新 prj.conf

根据新板型的硬件能力调整 `prj.conf`：

```kconfig
# 关闭不需要的功能以节省资源
CONFIG_LOG=n                    # 小 RAM 板型可关闭日志
CONFIG_SYS_LOG_LEVEL=0

# 或启用特定硬件驱动
CONFIG_GPIO=y
CONFIG_SPI=y
CONFIG_I2C=y
CONFIG_ADC=y
```

### 5.2 创建板型专用配置文件

为不同板型创建专用配置文件：

```
prj.conf                        # 默认配置
prj_nucleo_l4r5zi.conf          # L4 系列配置
prj_nucleo_h743zi.conf          # H7 系列配置（高性能）
prj_low_ram.conf                # 小 RAM 板型精简配置
```

构建时指定：

```bash
west build -b <board_name> -DCONF_FILE="prj.conf;prj_<board_name>.conf" .
```

### 5.3 调整模块依赖

某些模块可能依赖特定硬件，需在 `app_config.h` 中调整：

```c
// 根据板型启用/禁用模块
#if defined(CONFIG_BOARD_NUCLEO_L4R5ZI)
    #define APP_INIT_PRIO_MODULE_GPIO  85
#elif defined(CONFIG_BOARD_NUCLEO_H743ZI)
    #define APP_INIT_PRIO_MODULE_GPIO  85
    #define ENABLE_HIGH_PERF_MODULE    1
#else
    #define ENABLE_HIGH_PERF_MODULE    0
#endif
```

## 步骤 6：更新 CI/CD 配置

### 6.1 更新 GitHub Actions

编辑 `.github/workflows/ci.yml`，添加新板型到构建矩阵：

```yaml
jobs:
  build:
    strategy:
      matrix:
        board:
          - nucleo_l4r5zi
          - nucleo_h743zi    # 新增板型
          - custom_board
```

### 6.2 更新 GitLab CI

编辑 `.gitlab-ci.yml`：

```yaml
build_job:
  variables:
    BOARD: "nucleo_h743zi"  # 添加或修改板型
```

### 6.3 添加板型测试

如有硬件在环（HIL）测试，更新测试配置：

```yaml
test:
  stage: test
  variables:
    TEST_BOARD: "nucleo_h743zi"
  script:
    - west test -b ${TEST_BOARD}
```

## 步骤 7：验证与调试

### 7.1 构建验证

```bash
# 清理构建
west build -t pristine

# 构建新板型
west build -b <board_name> .

# 检查是否有编译错误
# 检查内存使用是否在限制内
```

### 7.2 查看内存使用

```bash
# 查看内存映射
arm-none-eabi-nm --size-sort build/zephyr/zephyr.elf
arm-none-eabi-size build/zephyr/zephyr.elf
```

### 7.3 烧录测试

```bash
# 烧录到新板型
west flash

# 如使用不同调试器，可能需要调整
west flash --runner jlink
west flash --runner pyocd
```

### 7.4 串口监控

```bash
# 打开串口监控
west console

# 指定串口号
west console --port COM3    # Windows
west console --port /dev/ttyUSB0  # Linux
```

### 7.5 调试常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 构建失败 | 板型未识别 | 检查 `BOARD` 名称是否正确 |
| 链接错误 | RAM/Flash 不足 | 减小堆栈大小，关闭非必要功能 |
| 运行异常 | 设备树配置错误 | 检查 `.overlay` 文件 |
| 无串口输出 | UART 未启用 | 检查 `CONFIG_UART_CONSOLE` 和设备树 |
| 烧录失败 | 调试器不匹配 | 调整 `--runner` 参数 |

## 常见问题

### Q1: 如何查找板型的准确名称？

```bash
# 在 Zephyr 目录下
cd %ZEPHYR_BASE%
west boards | grep <keyword>
```

### Q2: 自定义板型不被识别？

确保：
1. `CMakeLists.txt` 中启用了 `BOARD_ROOT`
2. 板型目录结构正确
3. `.yaml` 文件中的 `identifier` 与板型名一致

### Q3: 如何迁移现有配置到新板型？

1. 复制当前 `prj.conf` 和 `.overlay` 文件
2. 根据新板型规格调整内存配置
3. 修改设备树中外设的引脚定义
4. 逐步启用功能，验证稳定性

### Q4: 多板型如何管理配置？

推荐使用配置组合：

```
prj.conf                    # 公共基础配置
prj_common.overlay          # 公共设备树覆盖
prj_<board_name>.conf       # 板型特定配置
boards/<board_name>.overlay # 板型特定覆盖
```

构建时组合：

```bash
west build -b <board_name> -DCONF_FILE="prj.conf;prj_<board_name>.conf" .
```

### Q5: 如何验证板型迁移是否成功？

完成以下检查清单：

- [ ] 构建无编译错误
- [ ] 内存使用在板型限制内
- [ ] 烧录成功
- [ ] 串口有输出
- [ ] 基本功能（GPIO、UART）正常工作
- [ ] 项目特定功能正常
- [ ] CI/CD 通过
