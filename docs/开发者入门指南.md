## 从模板复制后的检查清单

将本仓库作为**新产品的工程模板**时，建议按下面清单逐项核对，避免路径、版本与 CI 长期漂移。

1. **west.yml**：`revision` 与团队使用的 Zephyr 版本一致（默认 **v3.6.0**，与 **[Zephyr版本与CI说明.md](Zephyr版本与CI说明.md)**、CI 镜像一致）；若使用公司内镜像，修改 `url`。
2. **zephyr_config.env**：从 **`zephyr_config.env.template`** 复制，填写 **`ZEPHYR_BASE`**、**`ZEPHYR_SDK_INSTALL_DIR`**；**不要提交**到 Git（仓库 **`.gitignore`** 已包含该文件）。若历史曾误提交过，需从索引中移除：`git rm --cached zephyr_config.env`（再提交一次）。
3. **CMake 工程名**：根目录 **`CMakeLists.txt`** 中 **`project(...)`** 改为你的产品名。
4. **版本与文档**：根目录 **`APP_VERSION`**、**`README.md`** 标题与产品说明按需更新。
5. **板型与 CI**：**`prj.conf`**、**`.github/workflows/ci.yml`** 中 ARM 构建矩阵的 **`board`** 与目标硬件或 CI 策略一致；文档中的示例板名（如 `nucleo_l4r5zi`）仅作演示，**以你实际 `BOARD` 与 CI 为准**。
6. **示例模块**：**`src/modules/example_*`** 可删除或改为业务模块；同步 **`CMakeLists.txt`** 与 **`app_main.c`** 中的注册与 Kconfig。
7. **可选钩子**：本地安装 **`pre-commit`**（见 **[参与贡献与代码规范.md](参与贡献与代码规范.md)**）可与 CI 中 **`pre-commit run --all-files`** 行为对齐。

## 快速开始

### 1. 环境配置

```bash
# Windows PowerShell
.\scripts\setup_env.ps1

# Linux/macOS
source scripts/setup_env.sh
```

### 2. 构建项目

```bash
# 构建 native_posix（测试用）
west build -b native_posix .

# 构建目标开发板（示例：按你手中的板名替换）
west build -b nucleo_l4r5zi .
```

### 3. 烧录和监控

```bash
# 烧录
west flash

# 串口（示例：需 pip install pyserial；COM 口与波特率按实际修改）
python -m serial.tools.miniterm COM3 115200
```

更完整的步骤与 Windows/Linux 差异见 **[烧录与调试快速指南.md](烧录与调试快速指南.md)**；排错见 **[常见问题与故障排除.md](常见问题与故障排除.md)**。

## 开发工作流

### 添加新模块

1. 创建模块文件 `src/modules/my_module.h/c`
2. 实现模块接口
3. 在 `CMakeLists.txt` 中添加源文件
4. 在 `app_main.c` 中注册模块

可选：模块间启动顺序若需按依赖排列，见 [模块系统详细使用说明.md](模块系统详细使用说明.md) 中的「运行时依赖」；多依赖示例见 `src/modules/example_module_multi_dep.c`。

### 添加事件类型

```c
// 在模块头文件中定义
#define EVENT_TYPE_MY_EVENT  100

// 注册事件类型
event_register_type(EVENT_TYPE_MY_EVENT, "my_event");

// 订阅事件
event_subscribe(EVENT_TYPE_MY_EVENT, my_callback, user_data, &id);

// 发布事件
event_publish_copy(EVENT_TYPE_MY_EVENT, EVENT_PRIORITY_NORMAL, &data, sizeof(data));
```

### 修改配置

- `prj.conf` - Zephyr 内核与应用合并配置（`CONFIG_*`）
- `Kconfig` - 本应用扩展的菜单项定义（根目录与 `src/modules/ipc_service/Kconfig`）
- `zephyr_config.env` - 本地路径配置
- `src/app/app_config.h` - 应用功能开关（`APP_CONFIG_*` 等宏，非 menuconfig）

**各配置项含义**：见 **[项目配置项说明.md](项目配置项说明.md)**（事件系统、模块管理器、系统服务、Thread IPC、`app_config` 补充说明）。

## 代码规范

### 格式化代码

```bash
# 使用 clang-format（仓库根目录 .clang-format）
clang-format -i src/**/*.c src/**/*.h
```

### pre-commit（可选）

```bash
pip install pre-commit
pre-commit install
pre-commit run --all-files
```

配置见仓库根目录 `.pre-commit-config.yaml`（含行尾空白、YAML 检查、clang-format）。

### 静态分析（clang-tidy）

仓库提供 `.clang-tidy`（可按项目收紧或放宽检查项）。需先配置 Zephyr 构建以生成 `compile_commands.json`，例如：

```bash
west build -b native_posix . -- -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
clang-tidy -p build src/core/event_system.c
```

无 `compile_commands.json` 时，仅能用 `clang-tidy ... -- -I...` 手动传头，易与 Zephyr 不一致，建议优先用 `-p build`。

### 命名约定

| 类型 | 命名风格 | 示例 |
|------|---------|------|
| 文件 | snake_case | `event_system.c` |
| 函数 | snake_case | `event_publish()` |
| 类型 | snake_case_t | `event_t`, `event_status_t` |
| 枚举值 | UPPER_CASE | `EVENT_OK`, `EVENT_ERR` |
| 宏 | UPPER_CASE | `EVENT_MAX_SUBSCRIBERS` |
| 变量 | lower_case | `subscriber_id` |
| 模块名 | snake_case | `module_manager` |

## 测试

### 运行单元测试

```bash
# 构建（native_posix）
west build -b native_posix tests/ --build-dir build_tests

# 运行 ztest 可执行文件（Linux/macOS）
./build_tests/zephyr/zephyr

# 或一步完成
west build -t run --build-dir build_tests
```

需已设置 `ZEPHYR_BASE` 或存在仓库根目录的 `zephyr_config.env`。

### 添加新测试

在 `tests/` 目录创建测试文件：

```c
#include <zephyr/ztest.h>
#include <被测试模块.h>

ZTEST(test_module, test_feature)
{
    // 测试代码
    zassert_equal(expected, actual, "错误消息");
}

ZTEST_SUITE(test_module, NULL, NULL, NULL, NULL, NULL);
```

## 发布

### 版本更新

1. 更新版本号：
   - `CMakeLists.txt`
   - `README.md`
   - `Doxyfile`
   - `CHANGELOG.md`

2. 提交更改：
```bash
git add .
git commit -m "chore: 准备发布 v1.0.0"
```

3. 创建标签：
```bash
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```

### 打包固件

```bash
# 打包发布
scripts/package_release.sh -v 1.0.0

# 批量构建
scripts/build_all.sh -c
```

**Windows 说明**：上述 **`*.sh`** 为 **Shell** 脚本。在 **PowerShell / CMD** 中通常需通过 **Git Bash**、**MSYS2** 或 **WSL** 执行；若仓库 **`scripts/`** 下提供了同名的 **`.ps1` / `.bat`**（如 **`build_all.bat`**），可优先使用。详见 **[脚本与工具说明.md](脚本与工具说明.md)**。

## 常见问题

### Q: 如何更改目标开发板？

A: 修改构建命令中的 `-b` 参数：
```bash
west build -b nucleo_l4r5zi . -p always
```

### Q: 如何添加新的 Kconfig 选项？

A: 在 `Kconfig` 文件中添加：
```kconfig
config MY_FEATURE
    bool "Enable my feature"
    default y
    help
      Enable this feature to...
```

然后在 `prj.conf` 中启用：
```
CONFIG_MY_FEATURE=y
```

### Q: 如何调试内存问题？

A: 使用内存统计和泄漏检测：
```c
sys_mem_dump_allocations(SYS_MEM_POOL_GENERAL);
uint32_t leaks = sys_mem_check_leaks(SYS_MEM_POOL_GENERAL);
```
